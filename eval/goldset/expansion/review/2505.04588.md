# 2505.04588 ZeroSearch: Incentivize the Search Capability of LLMs without Searching

> section_tag: 질의계획;검색시점;에이전트오케스트레이션;학습형RL  ·  survey_source: s1;s2;s3

## 초록 (한글)
효과적인 정보 탐색은 LLM의 추론·생성 능력 향상에 필수적이다. 최근 연구는 실제 환경에서 라이브 검색 엔진과 상호작용하며 강화학습(RL)으로 LLM의 검색 능력을 개선하는 것을 탐구해왔다. 이런 접근은 유망하지만 두 가지 큰 난제에 직면한다. (1) 통제 불가능한 문서 품질: 검색 엔진이 반환하는 문서 품질은 예측 불가능해 학습에 잡음과 불안정성을 도입한다. (2) 과도한 API 비용: RL 학습은 빈번한 롤아웃을 요구해 수십만 건의 검색 요청을 수반할 수 있어 막대한 비용이 들고 확장성을 심각히 제약한다. 이를 해결하기 위해, 학습 중 시뮬레이션 검색으로 LLM이 실제 검색 엔진을 사용하는 능력을 유도하는 새로운 RL 프레임워크 ZEROSEARCH를 도입한다. 본 접근은 경량 지도 미세조정(SFT)으로 LLM을 질의에 대해 유용한 문서와 잡음 문서 모두를 생성할 수 있는 검색 모듈로 변환하는 데서 시작한다. RL 학습 중에는 생성 문서 품질을 점진적으로 떨어뜨리는 커리큘럼 기반 롤아웃 전략을 사용해, 점점 어려워지는 검색 시나리오에 노출시켜 모델의 추론 능력을 점진적으로 이끌어낸다. 실험 결과 3B LLM을 검색 모듈로 써도 효과적으로 검색 능력을 유도하며, 7B 검색 모듈은 실제 검색 엔진과 비슷한 성능을, 14B 모듈은 오히려 능가했다. 또한 다양한 크기의 base/instruction 모델에 일반화되고 다양한 RL 알고리즘과 호환된다.

## Introduction (한글)
LLM은 다양한 과제에서 뛰어나지만, 사전학습 데이터 범위에 갇힌 정적 지식 탓에 환각이나 오래된 정보를 생성하기 쉬워 실용 신뢰성을 해친다. 따라서 외부 정보 접근이 필수적이다. 널리 쓰이는 접근인 RAG는 외부 지식을 생성 파이프라인에 통합한다. 초기 연구는 질의 생성·분해·다중 턴 검색을 안내하는 프롬프트 기반 전략에 집중했으나, 정교한 프롬프트 엔지니어링이 필요하고 모델 추론 능력에 큰 부담을 준다. 효율 개선을 위해 후속 연구는 SFT로 소형 LLM 성능을 높였고, MCTS 같은 테스트 시점 스케일링도 탐구했으나 상당한 계산 오버헤드가 따른다.

최근 RL은 추론·의사결정 능력 강화를 통해 LLM 성능을 더 개선하는 유망한 전략으로 떠올랐다. OpenAI-o1, DeepSeek-R1 같은 RL 기반 모델은 명시적 단계별 감독 없이 보상 주도 학습만으로 논리 추론에서 큰 향상을 보였다. 이 패러다임 안에서 여러 연구가 RL로 정책 모델을 학습해 관련 정보를 더 효과적으로 검색하게 했다. 대표 예로 Search-R1, R1-Searcher, ReSearch가 있다. 특히 DeepResearcher와 WebThinker는 Google 같은 상용 검색 엔진과의 라이브 상호작용을 도입해 실제 웹 검색에 가까운 환경에서 학습한다. 그러나 RL을 실제 검색 시나리오와 결합하면 (1) 통제 불가능한 문서 품질, (2) 과도한 API 비용이라는 난제가 생긴다.

이를 해결하기 위해, 실제 검색 엔진과 상호작용하지 않고 검색 전략을 학습하게 하는 ZEROSEARCH를 제안한다. 핵심 통찰은 LLM이 사전학습에서 방대한 세계 지식을 습득했고 질의가 주어지면 관련 문서를 생성할 수 있다는 점이다. 실제 검색 엔진과 시뮬레이션 LLM의 주된 차이는 반환 콘텐츠의 텍스트 스타일뿐이며, 경량 SFT로 작은 LLM도 검색 엔진 행동을 효과적으로 모사할 수 있다. API 비용 제거에 더해, LLM 기반 문서 생성의 중요한 장점은 문서 품질을 제어할 수 있다는 것이다. SFT 시 정답/오답으로 이어지는 문서를 프롬프트 설계로 구분해, 프롬프트의 몇 단어만 바꿔 유용/잡음 문서를 생성하도록 학습시킨다. 이 위에 커리큘럼 롤아웃 기제를 도입해 학습이 진행될수록 생성 문서 품질을 점진적으로 떨어뜨려, 정책 모델이 먼저 기본 형식·과제 요건을 배우고 점차 더 어렵고 잡음 많은 검색 시나리오에 적응하게 한다. ZEROSEARCH는 REINFORCE, PPO, GRPO 등 널리 쓰이는 RL 알고리즘과 매끄럽게 통합된다. (이하 기여 압축)

## 한 줄 핵심
실제 검색 엔진 대신 SFT로 만든 시뮬레이션 LLM이 유용·잡음 문서를 생성하게 하고, 그 품질을 점진적으로 떨어뜨리는 커리큘럼 RL로 LLM의 검색 능력을 API 비용 없이 길러내는 프레임워크.
