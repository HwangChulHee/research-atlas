# 2501.15228 Improving Retrieval-Augmented Generation through Multi-Agent Reinforcement Learning

## 초록 (한글)
검색증강생성(RAG)은 외부 지식을 대형언어모델에 통합하여 질의응답(QA) 과제에서 사실성을 높이고 환각을 줄이는 데 널리 활용된다. 표준 RAG 파이프라인은 질의 재작성, 문서 검색, 문서 필터링, 답변 생성 등 여러 구성요소로 이루어진다. 그러나 이들 구성요소는 보통 지도 파인튜닝을 통해 개별적으로 최적화되며, 이는 개별 구성요소의 목표와 정확한 답변 생성이라는 전체 목표 사이의 불일치를 초래할 수 있다. 최근 강화학습(RL)으로 특정 RAG 구성요소를 최적화하려는 시도가 있었으나, 이들은 종종 두 구성요소만 있는 단순 파이프라인에 초점을 두거나, 모듈 간 복잡한 상호의존성과 협력적 상호작용을 충분히 다루지 못한다. 이러한 한계를 극복하기 위해, 우리는 다중 구성요소를 가진 복잡한 RAG 파이프라인을 다중 에이전트 협력 과제로 다루기를 제안하며, 각 구성요소를 RL 에이전트로 간주한다. 구체적으로 우리는 MMOA-RAG(RAG를 위한 다중 모듈 공동 최적화 알고리즘)를 제시하며, 다중 에이전트 강화학습을 사용해 모든 에이전트의 목표를 최종 답변의 F1 점수 같은 통합 보상으로 조화시킨다. 다양한 QA 벤치마크 실험은 MMOA-RAG가 파이프라인 전반의 성능을 효과적으로 높이고 기존 베이스라인을 능가함을 보여준다. 또한 종합적 절제 연구는 개별 구성요소의 기여를 검증하고 MMOA-RAG가 다양한 RAG 파이프라인·벤치마크에 적응 가능함을 입증한다.

## Introduction (한글)
LLM은 질의응답, 정보검색, 추론, 평가 등 다양한 과제에 널리 적용되었으나, 사전학습 후 내부 지식을 즉시 갱신할 수 없어 오래되거나 조작된 응답을 생성하기 쉽다. 이를 해결하기 위해 RAG는 외부 지식원에서 관련 정보를 검색해 LLM의 생성 능력을 강화한다. 최근 RAG 시스템은 질의 재작성, 1차 검색, 재순위, 문서 전처리, 답변 생성 등 여러 상호연결 모듈로 이루어진 복잡한 파이프라인으로 구축되는 경우가 많다.

RAG 시스템의 복잡성은 최적화를 특히 어렵게 만든다. 표준 지도 파인튜닝(SFT)은 각 모듈을 인간 주석 데이터로 독립 최적화하나, 이는 개별 구성요소 목표와 전체 목표 사이의 불일치를 낳는다. 예를 들어 검색 모듈은 nDCG 같은 지표를 최적화하도록 인간 라벨 관련성 데이터로 훈련되지만, 문서 관련성과 응답 품질 사이의 단절을 다루지 못한다(관련성 높은 문서가 항상 정확한 답변에 기여하는 것은 아니다).

이를 위한 기존 RAG 종단간 최적화 연구는 주의 분포, 생성 확률, EM 반복 등으로 최종 출력의 보상을 중간 모듈로 전파하려 한다. 그러나 초기 접근법은 검색기·생성기 두 구성요소의 단순 파이프라인에 주로 초점을 두고, 다중 구성요소·풍부한 상호의존성을 가진 복잡 시스템 공동 최적화의 일반화 프레임워크를 제공하지 못한다. 더 최근 방법은 DPO·PPO 같은 알고리즘으로 모듈별 보상 필요를 없애려 하지만, 여전히 개별 RAG 모듈을 고립적으로 최적화하며 상호작용 구성요소 간 협력 역학을 충분히 모델링하지 못한다.

본 논문에서 우리는 MMOA-RAG를 제안한다. RAG 파이프라인의 각 중간 구성요소를 에이전트로 다루고, 최적화 과정을 협력적 다중 에이전트 강화학습(Co-MARL) 문제로 정식화하여 에이전트(모듈)들이 함께 최종 결과의 공유 보상을 최대화하도록 한다. 구체적으로 질의 재작성기, 고정 문서 검색기, 문서 선택기, 답변 생성기 네 핵심 모듈에 MMOA-RAG를 적용하고, 최종 보상을 정답 대비 F1 점수로 정의한다. 이를 위해 Multi-Agent PPO(MAPPO) 알고리즘을 활용한다. HotpotQA, 2WikiMultihopQA, AmbigQA에서 Llama-3-8B-Instruct 기반 실험으로 효과를 입증한다.

## 한 줄 핵심
RAG 파이프라인의 각 모듈(질의 재작성기·검색기·선택기·생성기)을 강화학습 에이전트로 보고, 최종 답변 F1을 공유 보상으로 MAPPO로 공동 최적화하는 다중 에이전트 RAG 방법이다.
