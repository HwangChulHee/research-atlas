# 2510.05691 DecEx-RAG: Boosting Agentic Retrieval-Augmented Generation with Decision and Execution Optimization via Process Supervision

> section_tag: 학습형RL  ·  survey_source: s3

## 초록 (한글)
에이전트형 검색 증강 생성(Agentic RAG)은 동적 검색과 적응적 워크플로우를 통해 복잡한 과제의 처리 능력을 향상시킨다. 최근 진전(예: Search-R1)은 결과 지도(outcome-supervised) 강화학습이 강한 성능을 보임을 입증했다. 그러나 이 접근은 비효율적 탐색, 희소한 보상 신호, 모호한 전역 보상 피드백 문제를 여전히 안고 있다. 이를 해결하기 위해 우리는 DecEx-RAG를 제안한다. 이는 RAG를 의사결정(decision-making)과 실행(execution)을 포함하는 마르코프 결정 과정(MDP)으로 모델링하고, 데이터 확장을 최적화하는 효율적 가지치기(pruning) 전략을 도입한다. 포괄적 과정 수준(process-level) 정책 최적화를 통해 DecEx-RAG는 LLM의 자율적 과제 분해, 동적 검색, 고품질 답변 생성 능력을 크게 향상시킨다. 실험 결과 DecEx-RAG는 6개 데이터셋에서 평균 절대 6.2% 성능 향상을 달성해 기존 베이스라인을 크게 능가했다. 또한 가지치기 전략은 데이터 구축 효율을 거의 6배 개선해 과정 지도 RAG 훈련의 효율적 해법을 제공한다.

## Introduction (한글)
최신 LLM은 뛰어난 문제 해결 능력을 보였지만, 정적 훈련 데이터의 본질적 한계로 인해 동적·실시간 문제에서 큰 병목을 보인다. 검색 증강 생성(RAG)은 외부 지식베이스나 검색 엔진을 결합해 이를 완화하는 유망한 해법이다.

최근 진전은 결과 지도 RL을 사용해 동적 검색을 추론 과정에 통합하여, LLM이 자율적으로 검색 도구를 호출하고 뛰어난 결과를 내게 한다. 그러나 이 접근은 본질적 한계가 있다. 첫째, 탐색 효율이 낮다 — LLM이 완전한 추론 사슬을 생성한 뒤에야 결과 기반 보상 피드백을 받기 때문. 둘째, 보상 신호가 희소하여 수렴에 더 많은 데이터와 훈련 스텝이 필요하다. 셋째, 전역 보상은 스텝 수준의 국소 성능을 반영하기 어려워 세밀한 최적화에 도전이 된다.

인간이 복잡한 문제를 풀 때 검색 도구를 활용하는 방식에서 영감을 얻어, 우리는 Agentic RAG를 위한 과정 지도 프레임워크 DecEx-RAG를 제안한다. 이 프레임워크는 RAG를 두 단계로 구성된 MDP로 모델링한다: 의사결정과 실행. 의사결정 단계는 종료 결정(중복 반복이나 조기 중단 방지)과 검색 결정(하위 질문 해결에 LLM 내부 지식을 쓸지 검색 도구를 쓸지)을 포함한다. 실행 단계는 결정 실행의 품질(예: 고품질 하위 질문)에 집중한다. 의사결정과 실행 단계를 구조적으로 분해함으로써, 프레임워크는 실행 품질과 결정 효율을 모두 고려하는 세밀한 과정 수준 지도를 가능하게 한다.

DecEx-RAG 프레임워크 내에서 중간 추론 스텝에 대한 지도 신호 부재는 검색 트리 확장 시 광범위한 탐색을 요구해 층 깊이에 따라 시간 복잡도가 지수적으로 증가한다. 이를 위해 효율적 가지치기 전략을 제안한다 — 검색 트리 각 층에서 결정의 서로 다른 실행에 대해 다중 롤아웃 시뮬레이션을 수행하고, 집계된 보상을 피드백 신호로 사용해 중복 분기를 동적으로 가지치기한다. 6개 오픈도메인 QA 데이터셋 실험에서 결과 지도 강베이스라인 Search-R1 대비 평균 6.3% 향상, 가지치기로 데이터 구축 효율 거의 6배 개선을 달성했다.

## 한 줄 핵심
RAG를 "의사결정+실행" 2단계 MDP로 보고 과정 수준 보상으로 학습하며, 검색 트리 가지치기로 과정 지도 데이터를 6배 효율로 만드는 에이전트형 RAG 프레임워크.
