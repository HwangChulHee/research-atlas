# 2601.07376 OpenTinker: Separating Concerns in Agentic Reinforcement Learning

> section_tag: RAG강화추론  ·  survey_source: s3

## 초록 (한글)
저자들은 LLM 에이전트의 강화학습(RL)을 위한 인프라 OpenTinker를 소개한다. 이는 알고리즘 설계, 실행, 에이전트-환경 상호작용 전반에 걸친 관심사 분리(separation of concerns)를 중심으로 구축된다. 단일체적(monolithic) 종단간 RL 파이프라인에 의존하는 대신, OpenTinker는 에이전트 학습 시스템을 명확한 추상화 경계를 가진 가볍고 조합 가능한 구성요소로 분해한다. 사용자는 에이전트, 환경, 상호작용 프로토콜을 지정하고, 추론과 학습은 관리형 실행 런타임에 위임된다. OpenTinker는 공유 자원 위에서 LoRA 기반 및 전체 파라미터 RL, 지도 미세조정(SFT), 추론을 포함한 학습·추론 워크로드를 관리하는 중앙 스케줄러를 도입한다. 또한 OpenTinker를 다중 에이전트 학습으로 확장하는 설계 원칙을 논의하고, 실용적 에이전트 학습 시나리오에서 프레임워크의 효과를 보이는 RL 사용 사례들을 제시한다.

## Introduction (한글)
LLM이 다단계 추론을 수행하고 외부 도구와 상호작용하는 에이전트로 진화하면서, RL은 SFT를 넘어 에이전트 행동을 개선하는 핵심 기제가 되었다. 그러나 실제로 RL을 에이전트 워크플로우에 적용하는 데는 긴 호흡(long-horizon)의 롤아웃, 이질적 실행 패턴, 대규모에서 추론과 학습을 조율할 필요성 때문에 상당한 시스템 차원의 난제가 있다.

최근 LLM용 RL 시스템들은 확장성·효율을 크게 개선했다. OpenRLHF는 고처리량 롤아웃 엔진과 PPO 스타일 RLHF용 공학 최적화로 실용적 분산 학습을 강조한다. HybridFlow는 RLHF를 데이터플로우 DAG로 정식화하고 복잡한 다단계 RL 파이프라인을 표현하는 하이브리드 제어 모델을 도입한다. AReaL은 롤아웃 생성과 모델 최적화를 별도 GPU 클러스터의 비동기 파이프라인으로 분리해 하드웨어 활용도를 크게 높인다. 더 최근의 Agent-Lightning은 에이전트 워크로드에 집중해 에이전트 런타임과 학습 백엔드를 분리, 기존 RL 프레임워크와의 최소 침습 통합을 가능케 한다. 이 시스템들은 RL 학습 워크로드 확장에 뛰어나지만, 에이전트 프로그래밍을 실행과 분리하는 지원이 제한적이고 보통 학습 인프라의 반복 배포·구성을 전제해 환경 간 재사용이 어렵다.

시스템 관점에서 두 가지 공백이 남는다. 첫째, 에이전트 환경과 상호작용 프로토콜이 재사용 가능한 일급(first-class) 추상으로 다뤄지는 경우가 드물다. 둘째, 에이전트 프로그래밍을 실행과 분리하는 지원이 제한적이다. 상용 시스템 Tinker는 인프라 관심사를 추상화하고 관리형 실행 환경을 제공해 이 방향의 가치를 입증하지만, 독점 시스템이라 설계 선택과 추상이 공개되지 않아 재현성·확장성이 제한된다. 본 논문은 에이전트-환경 명세, RL 알고리즘, 실행·자원 관리 전반에 명확한 관심사 분리를 강제하는 오픈소스 프레임워크 OpenTinker를 제시한다.
