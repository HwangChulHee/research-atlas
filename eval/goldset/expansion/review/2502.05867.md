# 2502.05867 Self-Training Large Language Models for Tool-Use Without Demonstrations

> section_tag: RAG강화추론  ·  survey_source: s1

## 초록 (한글)
대형언어모델(LLM)은 여전히 사실 부정확성과 계산 오류(환각, 수학적 추론 실수 포함)에 취약하다. 최근 연구는 이러한 단점을 완화하기 위해 LLM에 도구를 증강했으나, 큐레이션된 정답(gold) 도구 사용 시연을 요구하는 경우가 많다. 본 논문에서 우리는 LLM이 시연 없이 도구 사용을 학습할 수 있는지 조사한다. 첫째, 도구 활용을 안내하는 제로샷 프롬프팅 전략을 분석한다. 둘째, LLM 자신을 이용해 도구 사용 추적(trace)을 합성하는 자기훈련(self-training) 방법을 제안한다. 우리는 기존 질의응답(QA) 데이터셋(TriviaQA, GSM8K)으로 구성한 데이터셋에 대해 모델을 파인튜닝하는 지도 파인튜닝과 선호 파인튜닝 기법을 비교한다. 실험 결과 도구 사용은 롱테일 지식 과제에서 성능을 높인다(평가 전용 PopQA에서 3.7%). 그러나 TriviaQA, GSM8K, NQ-Open 등 다른 데이터셋에서는 혼재된 결과를 낳는다. 우리의 발견은 시연 없이 외부 도구를 LLM에 통합하는 것의 잠재력과 도전을 강조한다.

## Introduction (한글)
LLM은 많은 NLP 과제에서 SOTA 성능과 "창발적 능력"을 보였으며, 과제 지시(때로 소수 예시)만으로 파인튜닝 기반 방법에 필적하는 성능을 달성할 수 있다. 그럼에도 LLM은 내부 파라미터 지식에만 의존하면 부정확하거나 비사실적인 텍스트(환각)를 생성하거나 잘못된 계산을 수행할 수 있다. 이러한 단점에서 동기를 얻어, 우리는 계산기나 검색 엔진 같은 외부 도구로 LLM을 증강하여 추론·문제해결 능력을 강화하는 접근을 탐구한다.

LLM 도구 증강은 최근 활발한 연구 분야다. 현재 방법은 주로 두 접근을 따른다: (i) 대규모 모델의 인컨텍스트 학습 능력을 활용하는 프롬프팅, (ii) 도구 사용 예시 데이터셋으로 LLM을 훈련하는 파인튜닝(주로 SFT 중심). 이들 데이터셋은 보통 대규모 데이터에서 샘플링되거나 ChatGPT 같은 더 발전된 LLM에서 생성되는데, 자원 집약적이고 일반화가 어렵다.

또한 도구 사용 데이터셋만으로의 SFT는 차선일 수 있다. 도구 사용은 본질적으로 개방형(open-ended) 과제로, 특정 도구로 주어진 문제를 푸는 단일·유일한 해를 명시하는 오라클 추적이 없기 때문이다. 많은 도구가 기능이 겹치고 각 도구는 여러 방식으로 쓰일 수 있으며, 도구의 유용성은 LLM 자체에 의존한다. 이는 도구 사용의 개방형 특성을 더 잘 포착하는 방식으로 모델을 최적화하도록 이끌며, 선호 파인튜닝(PFT)이 일반적으로 도움이 된다. 동시기 연구가 수학 에이전트 등 특정 도메인에 PFT를 적용했으나, 일반 도구 사용 시나리오에서의 잠재력은 미탐구 상태다.

이러한 도전에서 영감을 받아, 우리는 "오라클 도구 사용 추적 없이 LLM에게 도구 사용을 가르칠 수 있는가?"라는 질문에 답한다. 첫째, LLM의 지시 따르기 능력을 활용하는 제로샷 프롬프팅을 연구하여 그래디언트 갱신 없는 도구 사용 성능 베이스라인으로 삼는다. 둘째, LLM 자신을 이용해 도구 사용 추적이 담긴 데이터셋을 합성하는 자기훈련 접근을 제안하고, 최종 출력 및 추가 기준에 기반한 필터링 전략으로 데이터 품질을 높인다. 그 후 SFT와 PFT 두 파인튜닝 목적을 탐구한다. 실험 결과 제로샷 프롬프팅만으로도 어느 정도 도구 사용이 가능하나 모델 규모·과제에 따라 성능 저하가 있을 수 있고, 제안한 자기훈련은 롱테일 지식 과제 PopQA에서 향상되나 다른 데이터셋에서는 혼재된 결과를 낳는다.

## 한 줄 핵심
정답 시연 없이 LLM 자신이 도구 사용 추적을 합성하고 출력 필터링 후 SFT/선호 파인튜닝으로 학습시켜, 검색·계산 같은 도구 사용을 가르치는 자기훈련 방법이다.
