# 2501.14342 Chain-of-Retrieval Augmented Generation

> section_tag: 추론워크플로우  ·  survey_source: s1

## 초록 (한글)
본 논문은 최종 답변을 생성하기 전에 관련 정보를 단계별로 검색·추론하는 o1형 RAG 모델 훈련 접근법을 소개한다. 기존 RAG 방법은 보통 생성 전에 단일 검색 단계를 수행하는데, 이는 불완전한 검색 결과로 인해 복잡한 질의 처리에 효과가 제한된다. 반면 우리가 제안하는 CoRAG(Chain-of-Retrieval Augmented Generation)는 진화하는 상태에 기반하여 모델이 질의를 동적으로 재구성하도록 한다. CoRAG를 효과적으로 훈련하기 위해, 우리는 거부 샘플링(rejection sampling)을 활용해 중간 검색 사슬을 자동 생성하여, 정답 최종 답변만 제공하는 기존 RAG 데이터셋을 증강한다. 테스트 시점에는 샘플링되는 검색 사슬의 길이와 개수를 제어하여 모델의 테스트 시점 연산을 확장하는 다양한 디코딩 전략을 제안한다. 다수 벤치마크 실험이 CoRAG의 효능을 입증하며, 특히 멀티홉 질의응답에서 강력한 베이스라인 대비 EM 점수 10점 이상 향상을 관찰했다. KILT 벤치마크에서 CoRAG는 다양한 지식집약적 과제에 걸쳐 새로운 SOTA를 달성한다. 또한 CoRAG의 스케일링 거동을 이해하기 위한 종합 분석을 제공하여, 사실적이고 근거 있는 기반 모델 개발을 위한 토대를 마련한다.

## Introduction (한글)
RAG는 대형 기반 모델과 독점 데이터원의 통합을 필요로 하는 기업 응용의 핵심 기술 중 하나로, 근거 있고 사실적인 응답 생성을 가능케 한다. 기반 모델은 대규모 데이터로 훈련되고 배포 후 동결되는데, 롱테일 사실 지식을 기억하지 못하거나 허위 주장을 환각하는 경우가 잦다. RAG는 검색된 정보로 생성을 증강해 이를 완화한다.

현대 RAG 시스템은 보통 검색-생성의 순차 파이프라인을 사용하며, 효과는 주로 검색 정보 품질에 의존한다. 검색 모델은 효율성을 위해 설계되며, 밀집 검색기는 보통 바이인코더 구조로 문서·질의를 고정 크기 벡터로 압축한다. 이 선택은 빠른 근사 최근접 이웃 탐색을 허용하지만 복잡 질의 처리에 대한 표현력을 제약한다. 또한 멀티홉 추론 과제에서는 처음에 무엇을 검색해야 할지 불분명한 경우가 많아, 점진적으로 진화하는 추론 상태에 기반해 결정해야 한다.

검색 품질의 병목을 깨기 위해, 우리는 현재 상태에 기반해 관련 정보를 동적으로 검색하고 후속 검색 단계를 계획하는 프레임워크를 제안한다. 테스트 시점에 검색 단계 수를 조정함으로써, 검색기가 유용한 정보를 산출하지 못할 때 질의의 여러 측면을 탐색하고 다양한 질의 재작성 전략을 시도할 수 있다. 이는 인간의 문제 해결 과정을 반영한다.

모델의 인컨텍스트 학습 능력에만 의존하거나 독점 모델로부터의 증류에 의존하는 대신, 우리는 단계별 검색을 명시적으로 훈련하는 것을 옹호한다. 이를 위해 거부 샘플링을 사용해 기존 RAG 데이터셋을 중간 검색 사슬로 증강하고, 오픈소스 언어모델을 표준 다음 토큰 예측 목적으로 파인튜닝한다. 스케일링 거동을 조사하기 위해 그리디 디코딩, best-of-N 샘플링, 트리 탐색 등 다양한 테스트 시점 디코딩 전략을 제안한다.

(요지 보존: 실험에서 멀티홉 QA에서 강력한 베이스라인을 크게 능가하고, KILT에서 거의 모든 과제에 새 SOTA를 달성하며, 과제 유형별로 다른 스케일링 거동을 보인다.)

## 한 줄 핵심
정답만 있는 RAG 데이터셋에 거부 샘플링으로 중간 검색 사슬을 만들어 단계별 검색·추론을 명시적으로 훈련하고, 테스트 시점 연산을 확장하는 o1형 멀티홉 RAG 방법이다.
