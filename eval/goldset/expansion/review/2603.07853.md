# 2603.07853 SynPlanResearch-R1: Encouraging Tool Exploration for Deep Research with Synthetic Plans

> section_tag: RAG강화추론  ·  survey_source: s3

## 초록 (한글)
연구 에이전트는 모델이 도구를 써서 웹에서 정보를 수집해 사용자 질의에 답하게 하며, 내부 추론과 도구 사용을 동적으로 번갈아 수행해야 한다. 이러한 능력은 원리상 검증 가능 보상 강화학습(RLVR)으로 학습될 수 있으나, 우리는 에이전트가 종종 조기 종료와 편향된 도구 사용 같은 빈약한 탐색 행동을 보임을 관찰한다. 그 결과 RLVR 단독으로는 향상이 제한적이다. 우리는 더 깊은 탐색을 장려하는 도구 사용 궤적을 합성하여 콜드스타트 지도 미세조정(SFT) 동안 탐색을 형성하고 이후 RL에 강한 초기화를 제공하는 프레임워크 SynPlanResearch-R1을 제안한다. 일곱 개의 멀티홉·오픈웹 벤치마크에서 SynPlanResearch-R1은 SOTA 기준선 대비 Qwen3-8B에서 최대 6.0%, Qwen3-4B에서 5.8% 성능을 향상했다. 도구 사용 패턴과 훈련 동역학에 대한 추가 분석이 이 이득의 근원을 조명한다.

## Introduction (한글)
LLM은 복잡한 추론, 의사결정, 외부 도구 상호작용에서 뛰어난 능력을 보였다. 이는 웹을 자율적으로 탐색하고 다양한 소스에서 정보를 검색·종합하여 복잡하고 지식 집약적인 질의에 종합 답변·보고서를 생성하는 연구 에이전트의 부상을 이끌었다.

연구 에이전트의 선구는 RAG로, 초기 시스템은 문서 임베딩을 생성 과정에 직접 투입했고 이후 시스템은 검색 문서를 맥락에 포함했다. 근본 난제는 언제 추가 맥락이 필요한지 판단하는 것이었는데, 저품질 검색이 모델을 산만하게 하고 효율을 떨어뜨릴 수 있음이 관찰되었기 때문이다. Self-RAG는 GPT-4 같은 강력한 모델에서 증류한 보상으로 모델이 검색 구절의 필요성·관련성을 성찰하도록 훈련해 이를 해결하려 했다. 그러나 이 방법은 여전히 매우 제한된 행동 공간에서 작동한다: 모델은 단일 턴 상호작용에서 검색 사용 여부만 결정할 뿐, 무엇을 읽고 언제 멈출지는 제어하지 못한다. 오늘날 연구 에이전트(딥 리서치 시스템)는 수십 라운드의 검색 질의를 문서·이미지 처리 도구와 짝지어 복잡한 요청을 다룬다.

다중 턴 도구 사용으로 연구 에이전트를 어떻게 효과적으로 훈련할까? 전문가 궤적을 이용한 모방 학습 외에, 검증 가능 보상 강화학습(RLVR)이 더 직접적인 대안을 제시한다. 이 온폴리시 패러다임은 최종 과제 보상(최종 답이 정답을 포함하는지)만으로 연구 에이전트를 최적화해 밀집 보상 모델 훈련을 없앤다. 과정은 도구 설명·원하는 행동으로 에이전트를 프롬프트하고, 기본 도구 사용·형식을 가르치는 콜드스타트 SFT로 얻은 웜스타트 정책에서 시작하는 경우가 많다. 이후 정책은 스칼라 결과 기반 보상만으로 정제된다.

그러나 우리는 RLVR로 훈련된 웹 연구 에이전트가 복잡한 질의를 푸는 효과적 도구 사용 궤적을 발견하지 못함을 발견했다. 이 실패는 보통 두 가지로 나타난다: (1) 너무 적은 검색 질의 후 추론을 조기 종료하고, (2) 단일 추론 사슬 내에서 여러 도구를 효과적으로 조합하지 못해 얕거나 단편적인 증거 수집에 그친다. 우리는 이를 RLVR의 온폴리시 특성 탓으로 본다: 학습이 에이전트 자신의 롤아웃에서 부트스트랩되므로, 약한 초기화가 탐색을 제한해 정책을 차선의 도구 사용 행동에 가둔다.

이 초기화 병목을 해결하기 위해 우리는 RL 전 콜드스타트 SFT를 위한 계획 유도 데이터 합성 프레임워크 SynPlanResearch-R1을 제안한다. 핵심 아이디어는 정답·잘 형성된 결과를 내면서 더 깊은 탐색을 장려하는 도구 사용 궤적을 합성하고, 이를 써서 이후 RL 최적화의 천장을 높이는 더 강한 초기 정책을 형성하는 것이다. ReAct 프레임워크 위에서, 가볍고 무작위화된 도구 계획을 생성해 궤적 생성을 안내하고, 대규모 추론 모델이 더 긴 추론 사슬과 풍부한 도구 상호작용을 내도록 프롬프트한다. 모델 추론을 억누르지 않으면서 계획 준수를 장려하기 위해 도구 의존적 단서를 도입해 행동 선택을 부드럽게 유도하고, 정답이며 형식 오류가 없는 궤적만 유지한다. RL 단계에서는 다중 턴 도구 기반 훈련의 난제를 다루는 두 가지 안정화 전략(턴/토큰 한도 초과 무효 궤적의 정책 손실 마스킹, JSON 스키마 위반 시 즉시 생성 종료)을 제안한다.

HotpotQA, 2WikiMultihopQA, MuSiQue, Bamboogle, GPQA, WebWalkerQA, GAIA 일곱 벤치마크에서 SynPlanResearch-R1은 일관된 이득을 보였고, 통제 기준선(콜드스타트 SFT의 바닐라 RL, 처음부터 RL) 대비 더 깊은 도구 사용 탐색과 초기 훈련의 높은 정책 엔트로피를 유지함을 확인했다.

## 한 줄 핵심
RLVR 연구 에이전트의 조기 종료·얕은 탐색 문제를, ReAct 기반 무작위 도구 계획으로 깊은 탐색 궤적을 합성해 콜드스타트 SFT 초기화를 강화하는 프레임워크.
