# 2603.01050 MM-DeepResearch: A Simple and Effective Multimodal Agentic Search Baseline

> section_tag: RAG강화추론  ·  survey_source: s3

## 초록 (한글)
우리는 명시적 추론·계획, 다중 도구 호출, 모달리티 간 정보 종합이 가능해 딥 리서치 과제를 수행하는 멀티모달 연구 에이전트를 개발하고자 한다. 그러나 이러한 에이전트 개발에는 세 가지 주요 난제가 있다: (1) 검색 집약적 멀티모달 QA 데이터의 희소성, (2) 효과적인 검색 궤적의 부재, (3) 온라인 검색 API를 사용한 훈련의 막대한 비용. 이를 해결하기 위해 첫째, 모달리티 내·간에서 시각·텍스트 노드를 모델링·연결하는 하이퍼그래프 기반 QA 생성 방법 Hyper-Search를 제안하여, 다양한 검색 도구 호출이 필요한 검색 집약적 멀티모달 QA 쌍을 생성한다. 둘째, 검색 관련 과제를 검색 도구 유형에 따라 여러 범주로 분해하고 각 도구별 전문가(tool expert)를 각각 최적화한 뒤, 도구 전문가들을 재조합하여 트리 탐색으로 검색 궤적을 공동 탐색해 복잡한 과제를 다양한 도구로 성공적으로 푸는 궤적을 생성하는 DR-TTS를 도입한다. 셋째, 여러 검색 도구를 지원하는 오프라인 검색 엔진을 구축해 값비싼 온라인 검색 API 없이 에이전트형 강화학습을 가능하게 한다. 이 세 설계로 강력한 멀티모달 딥 리서치 에이전트 MM-DeepResearch를 개발했으며, 광범위한 결과가 여러 벤치마크에서의 우월성을 보인다.

## Introduction (한글)
최근 추론 멀티모달 LLM(MLLM)은 중간 추론 과정을 명시적으로 생성하여 복잡한 과제 해결에서 뛰어난 능력을 보였다. 그럼에도 기존 모델은 용량이 제한된 매개변수와 고정·유한한 지식에 근본적으로 제약되어, 내재 지식을 넘는 질문, 특히 정보 집약적이고 열린 세계 과제를 다루는 능력이 제한된다.

이 한계를 완화하기 위해 초기 방법은 주로 RAG나 프롬프트 기반 접근처럼 사전 정의된 정보 검색 워크플로우를 채택했다. 이들은 사전 정의된 파이프라인에서 외부 정보를 맥락으로 검색한 뒤 추론을 시작한다. 그러나 이러한 접근은 검색을 추론 과정과 분리하여, 모델의 추론 상태가 진화함에 따라 검색 전략을 반복적으로 적응시키는 능력이 없어 검색 능력과 일반화가 제한된다.

우리는 명시적 추론·계획, 다중 도구 호출, 모달리티 간 정보 종합을 포함한 에이전트형 검색 능력을 갖춘 멀티모달 딥 리서치 모델을 개발하고자 하며, 세 가지 근본 난제를 관찰한다: (1) 검색 집약적 멀티모달 QA 데이터의 희소성 — 다중 턴 검색·다중 도구 호출을 포함한 공개 데이터셋이 매우 제한적이라 감독이 부족하다. (2) 효과적인 검색 궤적의 부재 — 전통적 프롬프트 기반 궤적 합성은 주로 단일 턴 검색용으로 설계되어 다중 턴 시나리오에 부적합하다. (3) 온라인 검색 API 훈련의 막대한 비용 — 대부분의 방법이 SerpAPI, Jina 같은 온라인 API에 의존해 훈련 1회당 수천 달러가 들어 광범위한 실험을 제약한다.

이를 해결하기 위해 세 가지 상호보완적 기법을 제안한다. 첫째, Hyper-Search는 웹 시각·텍스트 콘텐츠를 노드로 공동 모델링하고 하이퍼에지로 노드 간 본연의 관계를 포착해 복잡한 다중 소스·멀티모달 정보 의존성을 구조적으로 표현한다. 이 하이퍼그래프를 바탕으로 둘 이상의 노드를 하나 이상의 하이퍼에지로 엮어, 각 질문이 본질적으로 다중 턴·다중 유형 검색 도구 호출을 요구하도록 검색 집약적 멀티모달 QA 쌍을 구성한다.

둘째, Decompose–Recompose Tool Tree Search(DR-TTS)를 도입한다. DR-TTS는 검색 관련 과제를 검색 도구 유형에 따라 여러 범주로 먼저 분해하고, 각 전문가가 단일 도구를 숙달하도록 강화학습으로 전문가들을 각각 훈련한다. 이 분해 설계는 학습 복잡도를 단순화하고 도구별 숙련도를 높인다. 이어 도구 전문가들을 재조합해 트리 탐색으로 유효 검색 궤적을 공동 탐색·식별하여 SFT용 고품질 궤적을 생성한다. 단일 모델로 직접 궤적을 생성하는 것보다 도구 사용 편향을 완화하고 탐색 다양성·성공률을 높인다.

셋째, 위 두 설계 위에 정보 기반·지식 기반 검색을 모두 지원하는 오프라인 검색 엔진을 구축한다. 정보 기반 도구는 사전 구축 멀티모달 코퍼스에서 시각·텍스트 맥락을 검색하고, 지식 기반 도구는 복잡한 질의를 받아 모델이 생성한 지식 집약적 콘텐츠를 반환한다. 온라인 엔진 대비 더 빠른 도구 응답과 줄어든 상호작용 훈련 시간을 제공하며, 무엇보다 값비싼 온라인 API 비용을 회피한다.

이 설계들로 강력한 멀티모달 딥 리서치 에이전트 MM-DeepResearch를 훈련한다. Hyper-Search로 생성한 데이터, DR-TTS로 합성한 궤적, 오프라인 검색 엔진으로 훈련된 MM-DeepResearch는 온라인 API로 평가 시 여러 벤치마크에서 우월한 성능을 달성하며, 비싼 온라인 API로 훈련된 이전 에이전트를 능가한다. 기여는 네 가지(Hyper-Search, DR-TTS, 오프라인 검색 엔진, 광범위한 실험)다.

## 한 줄 핵심
하이퍼그래프 QA 생성(Hyper-Search) + 도구별 전문가 분해·재조합 트리 탐색(DR-TTS) + 오프라인 검색 엔진으로, 비싼 온라인 API 없이 멀티모달 딥 리서치 에이전트를 훈련한 베이스라인.
