# 2604.04651 Search, Do not Guess: Teaching Small Language Models to Be Effective Search Agents

> section_tag: 검색시점  ·  survey_source: s3

## 초록 (한글)
검색 도구를 갖춘 에이전트는 지식 집약적 과제의 효과적 해법으로 부상했다. 대형 언어 모델(LLM)은 강력한 추론 능력을 보이지만 높은 계산 비용 때문에 검색 에이전트로의 실제 배포가 제약된다. 그래서 최근 연구는 LLM의 에이전트적 행동을 소형 언어 모델(SLM)로 증류하는 데 집중해 왔다. 복잡한 멀티홉 추론 과제에 대한 종합 평가를 통해, SLM은 파라미터 지식이 적음에도 검색 도구를 덜 자주 호출하고 환각에 더 취약하다는 점을 발견했다. 이를 해결하기 위해 SLM이 검색된 증거에 근거해 신뢰성 있게 검색·답변하도록 명시적으로 훈련하는 경량 파인튜닝 기법 Always-Search Policy(ASP)를 제안한다. LLM으로부터의 에이전트 증류 대비 Bamboogle에서 17.3점, HotpotQA에서 15.3점 향상되어 여러 벤치마크에서 LLM 수준 결과를 달성한다. 추가 분석에서 SLM의 적응형 검색 전략은 성능을 저하시키는 경우가 많아, 신뢰성 있는 추론에는 일관된 검색 행동이 필요함을 강조한다.

## Introduction (한글)
언어 모델 기반 검색 에이전트는 검색 질의를 반복적으로 발행하고, 검색된 증거 위에서 추론하며, 중간 결과에 따라 이후 행동을 조정한다. Search-o1 같은 에이전트는 복잡한 정보 탐색 문제 해결에 강한 잠재력을 보였다. 그러나 기존 검색 에이전트는 대형 LLM(보통 7B 이상)에 의존해 지연·예산 제약이 있는 실제 배포를 어렵게 한다. 효율성 때문에 SLM(4B 미만)이 선호되는데, 이것이 "에이전트적 검색 능력을 어떻게 SLM에 효과적으로 증류할 것인가"라는 중심 질문의 동기다.

먼저 Qwen3-1.7B를 멀티홉 QA에서 평가해 핵심 실패 모드인 "파라미터 환각"을 식별한다. SLM은 제한된 파라미터 지식에 과의존해 추측성 답을 생성하는 경향이 있다. 또한 LLM(Qwen3-32B) 궤적을 단순 증류하는 것은 미미한 개선(F1 50.6→53.2)만 주는데, LLM 궤적이 SLM에는 없는 파라미터 지식에 암묵적으로 의존하는 추론 단계를 담고 있기 때문이다. 이를 해결하기 위해, 훈련 중 검색 행동을 명시적으로 제약하는 증류 패러다임 ASP를 제안한다. ASP는 학생 모델이 답하기 전에 항상 외부 검색으로 필수 정보를 검색·근거화하도록 강제하여 증거 기반 추론을 우선시하고 추측성 추론을 억제한다. 그 결과 F1을 53.2→70.6으로 끌어올려 교사와의 격차를 2.5점으로 줄인다. 나아가 SLM이 파라미터 지식 신뢰도에 따라 검색 여부를 적응적으로 결정할 수 있는지 조사했는데, 가장 자신 있는 5% 질의만 자가응답해도 큰 성능 저하가 발생했다. SLM은 일관되고 강제된 검색에서 이득을 보며 ASP가 가장 효과적인 정책으로 나타난다.

## 한 줄 핵심
소형 언어 모델이 추측 대신 항상 외부 검색으로 근거를 확보하도록 강제 훈련(ASP)해 파라미터 환각을 줄이고 LLM 수준의 멀티홉 검색 성능에 도달시킨다.
