# 2505.22648 WebDancer: Towards Autonomous Information Seeking Agency

> section_tag: 학습형RL  ·  survey_source: s2

## 초록 (한글)
복잡한 실세계 문제를 다루려면 심층 정보 탐색과 다단계 추론이 필요하다. Deep Research로 대표되는 최근 에이전트 시스템의 진전은 자율적 다단계 연구의 잠재력을 보여준다. 본 연구는 데이터 중심·학습 단계 관점에서 end-to-end 에이전트형 정보 탐색 에이전트를 구축하는 일관된 패러다임을 제시한다. 본 접근은 네 핵심 단계로 구성된다. (1) 브라우징 데이터 구축, (2) 궤적 샘플링, (3) 효과적 cold start를 위한 지도 미세조정, (4) 일반화 강화를 위한 강화학습. 이 프레임워크를 ReAct 기반 웹 에이전트 WebDancer로 구현한다. 도전적인 정보 탐색 벤치마크 GAIA와 WebWalkerQA에서의 실증 평가는 WebDancer의 강한 성능을 보이며 본 학습 패러다임의 효능을 부각한다. 에이전트 학습에 대한 추가 분석은 더 유능한 에이전트 모델 개발을 위한 가치 있는 통찰과 체계적 경로를 제공한다.

## Introduction (한글)
웹 에이전트는 실세계 웹 환경을 인지하고 결정을 내리며 인간 같은 과제를 수행하는 자율 시스템이다. ChatGPT Deep Research, Grok DeepSearch 같은 최근 시스템은 end-to-end 강화학습 학습을 통해 강한 심층 정보 탐색 능력을 보였다. 커뮤니티의 기존 접근은 두 유형으로 나뉜다. (i) 프롬프트 엔지니어링 기법으로 LLM/LRM을 직접 안내해 복잡 과제를 수행, (ii) SFT나 RL로 웹 에이전트에 검색·브라우저 능력을 통합. 첫 학습 불요 방법은 추론 모델이 가능케 하는 추론 능력을 효과적으로 활용하지 못한다. 후자는 SFT/RL로 일부 정보 탐색 능력을 내재화하지만, 학습·평가 데이터셋이 비교적 단순해 실세계 난제를 포착하지 못한다(예: 2Wiki는 이미 80% 이상). 또한 현 SFT/RL 패러다임은 정보 탐색 행동의 잠재력을 충분·효율적으로 활용하지 못한다. 자율 정보 탐색 능력 구축은 (1) 다양한 사용자 의도와 풍부한 상호작용 맥락을 반영하는 고품질·세분 브라우징 데이터 확보, (2) 장기 추론·과제 분해를 지원하는 신뢰할 궤적 구성, (3) OOD 웹 환경·복잡 상호작용·장기 목표에 걸쳐 견고한 행동을 부여하는 확장·일반화 가능 학습 전략 설계라는 난제들을 다뤄야 한다.

이를 위해, Deep Research 같은 웹 에이전트를 처음부터 구축하는 법을 탐구한다. Deep Research 같은 에이전트 모델은 추론·행동 단계가 교차된 시퀀스를 생성하며, 각 행동은 외부 환경과 상호작용하는 도구를 호출한다. 상호작용에서의 관찰이 이후 추론·행동을 안내하며 과제 완료까지 이어지고, 이 과정은 end-to-end 도구 증강 학습으로 최적화된다. ReAct 프레임워크는 추론과 행동을 긴밀히 결합해 상호작용 설정에서 효과적 학습·일반화를 촉진하므로 가장 적합한 패러다임이다.

데이터 중심 관점에서, 웹 QA 데이터 구축은 SFT든 RL이든 웹 에이전트 구축에 핵심이다. 널리 쓰이는 QA 데이터셋은 단일·소수 턴 검색으로 풀리는 얕은 문제가 많고, 도전적 웹 기반 QA 데이터셋은 보통 테스트/검증 셋만 있고 크기가 작다(GAIA 466, WebWalkerQA 680, BrowseComp 1,266). 따라서 고품질 데이터셋 자동 합성이 중요하다. 두 방식으로 합성한다. 1) 웹 페이지를 크롤링해 클릭 행동으로 정보 획득이 필요한 심층 질의를 구성(CRAWLQA), 2) 쉬운-어려운 QA 쌍 합성을 강화해 단순 질문을 복잡하게 변환(E2HQA). 학습 단계 관점에서, 기존 SFT/off-policy RL은 일반화 문제가 있고, on-policy RL 직접 적용은 다중 도구 설정에서 초기 단계가 주로 도구 사용 학습에 쏠린다. 이에 rejection sampling fine-tuning(RFT)과 후속 on-policy RL을 결합한 2단계 접근을 채택한다. 궤적 샘플링에서는 행동 공간을 search·click 두 도구로 제한하고, 강한 instruction LLM(Short-CoT)과 LRM(Long-CoT) 두 프롬프트 전략으로 rejection sampling 궤적을 생성한다. RL 단계에서는 동적 샘플링 기제로 SFT에서 덜 활용된 QA 쌍을 효과적으로 활용하는 DAPO 알고리즘을 채택한다. (이하 기여 압축)

## 한 줄 핵심
브라우징 데이터 합성 → 궤적 샘플링 → SFT cold start → RL의 4단계 파이프라인으로 ReAct 기반 자율 웹 정보 탐색 에이전트(WebDancer)를 처음부터 구축하는 체계적 레시피.
