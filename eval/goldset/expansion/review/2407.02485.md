# 2407.02485 RankRAG: Unifying Context Ranking with Retrieval-Augmented Generation in LLMs

> section_tag: 정보필터링  ·  survey_source: s2

## 초록 (한글)
대규모 언어 모델(LLM)은 검색 증강 생성(RAG)에서 보통 검색기가 준 상위 k개 문맥을 사용한다. 본 연구는 단일 LLM을 문맥 랭킹과 답변 생성이라는 두 목적으로 동시에 지시 미세조정(instruction-tune)하는 새로운 프레임워크 RankRAG를 제안한다. 특히 지시 미세조정된 LLM은 학습 혼합(training blend)에 소량의 랭킹 데이터를 추가하는 것만으로 놀랍도록 잘 작동하며, 대량의 랭킹 데이터만으로 미세조정한 동일 LLM을 포함한 기존 전문 랭킹 모델을 능가한다. 생성 측면에서는 GPT-4-0613, GPT-4-turbo-2024-0409, 그리고 RAG 벤치마크 SOTA 오픈소스 모델 ChatQA-1.5 등 강력한 기준선과 비교한다. 구체적으로 Llama3-RankRAG는 아홉 개 지식 집약 벤치마크에서 Llama3-ChatQA-1.5와 GPT-4 모델을 크게 능가한다. 또한 생의학 데이터로 지시 미세조정하지 않고도 다섯 개 생의학 RAG 벤치마크에서 GPT-4에 견줄 만한 성능을 보여, 새 도메인으로의 뛰어난 일반화 능력을 입증했다.

## Introduction (한글)
RAG는 모델 가중치를 바꾸지 않고 LLM을 롱테일 지식, 최신 정보, 특정 도메인/과제에 맞추는 데 널리 쓰인다. 보통 밀집 임베딩 기반 검색기가 질문에 대해 상위 k개 청크 문맥을 검색하고, LLM이 이를 읽어 답을 생성한다. 그러나 현재 RAG 파이프라인은 한계가 있다: i) LLM은 긴 문맥 창이 있어도 너무 많은 청크(예: top-100)를 잘 읽지 못하며, 효율뿐 아니라 정확도 면에서도 더 짧은 top-k(예: 5,10)가 더 낫다. ii) k가 작을 때 관련 내용의 높은 재현율(recall)을 보장하는 메커니즘이 필요한데, 검색 모델만으로는 부족하다. 실제로 질문-후보 문맥을 교차 인코딩하는 별도 랭킹 모델이 상위 N개 후보(N≫k)에서 가장 관련 높은 top-k를 얻는 데 밀집 임베딩 검색기보다 잘 작동할 수 있다. iii) 그러나 전문 랭킹 모델의 제로샷 일반화 능력은 다재다능한 LLM 자체보다 상대적으로 제한적이다. 이를 바탕으로 저자들의 목표는 단일 LM으로 고재현율 문맥 추출과 고품질 생성을 모두 달성하는 RAG 지시 튜닝 파이프라인 설계다. 지시 미세조정된 LLM은 관련 문맥에서 답을 추출하는 강한 능력을 보이는데, 이는 문맥 청크가 질문에 관련 있는지(즉 답 생성에 유용한지) 판단하는 "이중 능력(dual capability)"으로 볼 수 있고, 두 능력이 상호 강화된다고 가정한다. 이에 RankRAG를 제안한다: 단일 LLM을 문맥 랭킹과 답변 생성 둘 다에 지시 튜닝하며, 문맥 풍부 QA·검색 증강 QA·랭킹 데이터셋으로 지시 튜닝 데이터를 확장해 무관 문맥을 걸러내는 능력을 강화한다. 추론 시 LLM이 먼저 검색 문맥을 재정렬(rerank)한 뒤 정제된 top-k(예: 5)로 답을 생성한다. 기여: (1) 단일 LLM을 문맥 랭킹+답변 생성에 동시 지시하는 새 프레임워크 (2) 소량 랭킹 데이터 추가만으로 10배 많은 랭킹 데이터로 튜닝한 LLM도 능가 (3) ChatQA-1.5 등 강한 기준선 대비 9개 일반·5개 생의학 벤치마크에서 큰 마진 우위.

## 한 줄 핵심
단일 LLM을 "문맥 재랭킹"과 "답변 생성" 양쪽에 지시 미세조정해, 검색된 문맥을 LLM 스스로 재정렬한 뒤 상위 k개로 답하게 하여 별도 랭킹 모델 없이 RAG 성능을 높이는 방법.

## builds_on 초안 추천  *(사람이 원문으로 검증할 초안 — 정답 아님)*
- RAG (Retrieval-Augmented Generation) — 근거: "Retrieval-augmented generation (RAG) (Lewis et al., 2020; Izacard & Grave, 2021; ...) is a widely used technique" — RankRAG는 표준 RAG 파이프라인(검색→top-k 읽기→생성)에 LLM 기반 재랭킹 단계를 통합해 확장하는 후속.
- ChatQA-1.5 — 근거: "ChatQA-1.5, an open-sourced model with the state-of-the-art performance on RAG benchmarks" 및 "Llama3-RankRAG-8B and Llama3-RankRAG-70B outperforms Llama3-ChatQA-1.5" — RankRAG가 따르는 지시 튜닝 RAG 모델 계열로 보이나, 본문상 주로 비교 baseline 성격이 강해 계보 여부는 사람 검증 요망.

(주의: GPT-4 류는 비교 baseline로 제외. Llama3는 substrate 백본으로 제외. 별도 랭킹/검색 모델(DPR 등)은 RankRAG가 대체하려는 형제로 제외.)
