# 2603.09203 Evaluate-as-Action: Self-Evaluated Process Rewards for Retrieval-Augmented Agents

> section_tag: 학습형RL  ·  survey_source: s3

## 초록 (한글)
검색증강 에이전트는 외부 증거를 질의할 수 있지만, 다단계 추론에서의 신뢰성은 여전히 제한적이다: 잡음 섞인 검색이 멀티홉 QA를 탈선시킬 수 있고, 결과 전용(outcome-only) 강화학습은 중간 단계를 최적화하기에 너무 거친 신용 신호만 제공한다. 우리는 암묵적 검색 품질 평가를 명시적 행동으로 변환하고, 각 검색 직후 구조화된 평가 점수가 따르도록 결합된 Search-to-Evaluate 프로토콜을 강제하여 상호작용 궤적에 정렬된 프로세스 신호를 산출하는 EVALACT(Evaluate-as-Action)를 제안한다. 이 신호를 활용하기 위해, 평가 점수에 따라 세그먼트 수준에서 어드밴티지를 재척도화하여 신뢰할 만한 세그먼트를 강조하고 불확실한 세그먼트는 보수적으로 갱신하는 GRPO 기반 최적화 방법 PCAR(Process-Calibrated Advantage Rescaling)를 도입한다. 일곱 개 오픈도메인 QA 벤치마크 실험에서 EVALACT는 최고 평균 정확도를 달성했고 멀티홉 과제에서 이득이 가장 컸으며, 절제 연구는 명시적 평가 루프가 주된 향상을 이끌고 PCAR가 일관된 추가 이득을 제공함을 확인했다.

## Introduction (한글)
LLM 에이전트는 자동 추론을 수동적 응답 생성에서 자율적 문제 해결로 옮겨, 모델이 계획하고 외부 도구와 상호작용하며 다단계 궤적에 걸쳐 믿음을 반복 정제하게 했다. RAG는 외부 증거에 결정을 접지하여 이 능력을 더 확장해, 매개변수 지식의 한계를 넘는 오픈도메인 QA를 가능하게 한다.

그러나 질의가 단일홉 사실에서 멀티홉 서사로 옮겨가면서, 중심 병목은 도구 접근 자체가 아니라 긴 호흡·잡음에 취약한 상호작용 시퀀스에 걸쳐 증거를 항해·검증·종합하는 능력이 된다.

상당한 진전에도 신뢰할 만한 중간 추론 보장은 여전히 핵심 난제다. 검색·추론을 번갈아 하는 프롬프팅 방법부터 Search-R1 같은 RL 기반 검색 에이전트, AutoRefine 같은 정제 프레임워크까지, 기존 에이전트 기준선은 여전히 잡음 억제와 자기 교정을 주로 암묵적 내부 추론에 의존한다. 이 패러다임은 두 근본 한계를 겪는다. 첫째, 오류 전파: 증거 검증을 위한 명시적·즉각적 메커니즘 없이는 단일 무관 문서가 다운스트림 추론을 탈선시켜 멀티홉에서 되돌릴 수 없는 궤적 표류를 일으킨다. 둘째, 거친 신용 할당: PPO 기반 RLHF, GRPO 같은 결과 보상 사후 훈련을 포함한 표준 RL 최적화는 보통 최종 답 정확도에 묶인 희소 신호에 의존한다. 이러한 결과 전용 감독은 긴 궤적 내 유익한 검색 단계를 중복·오도하는 행동과 구별하지 못해, 최적화기가 궤적 전체를 거의 균일하게 강화·처벌하고 표본 효율을 떨어뜨리며 과제 복잡도가 커질수록 성능 포화를 일으킨다.

이를 해결하기 위해 우리는 에이전트의 암묵적 검색 품질 자기 평가를 명시적이고 정책이 선택 가능한 행동으로 변환하는 RL 프레임워크 EVALACT를 도입한다. EVALACT는 엄격히 결합된 search-then-evaluate 프로토콜을 강제한다: 각 Search 행동 뒤에 반드시 Evaluate 행동이 따라와 검색 증거의 유용성을 반영하는 구조화된 자기 평가 점수를 산출한다. 이 설계는 위 두 한계를 직접 다룬다. 추론 시에는 평가 출력이 비생산적 분기의 조기 가지치기를 돕는 실행 가능한 제어 신호를 제공해 외부 오라클 감독 없이 오류 전파를 줄인다. 훈련 시에는 밀집·궤적 정렬 프로세스 신호를 산출해 중간 신뢰성을 직접 최적화 가능하게 하고 더 세밀한 신용 할당을 가능하게 한다.

이 프로세스 신호를 효과적으로 활용하기 위해, GRPO 위에 구축된 PCAR를 추가 제안한다. 단일 궤적 수준 어드밴티지를 모든 토큰에 방송하는 대신, PCAR는 단계별 자기 평가 점수로 세그먼트 수준 갱신을 조절해 신뢰할 만한 진전 단계의 그래디언트를 증폭하고 불확실한 세그먼트에는 보수적 갱신을 적용한다. 이는 비싼 인간 주석 프로세스 보상 모델 없이 프로세스 수준 안내를 제공하며, 검색 행동을 명시적으로 겨냥하지 않는 기존 검증 지향 감독을 보완한다.

기여: (1) 암묵적 검색 품질 평가를 명시적 Evaluate 행동으로 변환하고 Search→Evaluate 프로토콜을 강제하는 EVALACT, (2) 단계별 자기 평가 점수로 신용 할당을 정제·안정화하는 GRPO 기반 PCAR, (3) 일곱 벤치마크·두 백본 규모에서 최고 평균 성능과 멀티홉 강한 이득, 절제로 평가 루프가 주된 향상을 차지함을 입증.

## 한 줄 핵심
검색마다 즉시 자기 평가 점수를 내는 명시적 Evaluate 행동(Search→Evaluate 프로토콜)을 도입하고, 그 점수로 세그먼트별 어드밴티지를 재척도화(PCAR)해 RAG 에이전트의 오류 전파·거친 신용 할당을 푸는 RL 프레임워크.
