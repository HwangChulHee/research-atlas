## 초록 (한글)
우리는 자기학습(self-learning)이 인간이 큐레이션한 데이터셋이나 사전 정의된 규칙 기반 보상에 의존하지 않고 LLM 기반 에이전트를 확장할 수 있는지를 연구한다. 검색 에이전트 환경에서의 통제된 실험을 통해, 확장 가능한 에이전트 훈련의 두 가지 핵심 결정 요인을 식별한다: 보상 신호의 출처와 에이전트 과제 데이터의 규모. 우리는 생성형 보상 모델(GRM)의 보상이 경직된 규칙 기반 신호보다 오픈도메인 학습에 우수하며, GRM을 정책과 공진화(co-evolve)시키면 성능이 더 향상됨을 발견한다. 또한 에이전트 과제 데이터의 양을 늘리면 — 합성 생성된 것이라도 — 에이전트 능력이 크게 향상된다. 이 통찰을 바탕으로 우리는 Agentic Self-Learning(ASL)을 제안한다. 이는 공유된 도구 환경과 LLM 백본 안에서 과제 생성, 정책 실행, 평가를 통합하는 완전 폐루프(closed-loop) 다중 역할 강화학습 프레임워크다. ASL은 프롬프트 생성기(Prompt Generator), 정책 모델(Policy Model), 생성형 보상 모델(GRM)을 조율해 더 어려운 과제 설정, 더 날카로운 검증, 더 강한 해결의 선순환을 형성한다. 실험적으로 ASL은 라운드마다 꾸준한 향상을 제공하며, 정체되거나 퇴화하는 강력한 RLVR 베이스라인(예: Search-R1)을 능가하고, 라벨 없는 데이터 조건에서도 계속 개선되어 우수한 샘플 효율과 견고성을 보인다. 또한 GRM 검증 역량이 주된 병목임을 보인다 — 고정되면 보상 해킹을 유발해 진전을 멈추게 하며, 진화하는 데이터 분포에서 GRM을 지속 훈련하면 이를 완화하고, 후반에 소량의 실제 검증 데이터를 주입하면 성능 상한을 끌어올린다.

## Introduction (한글)
LLM 기반 자율 에이전트는 초기 프롬프트 엔지니어링 파이프라인에서 자기개선이 가능한 RL 훈련 시스템으로 빠르게 진화했다. ReAct 스타일 추론-행동 프레임워크와 반성적 방법 같은 초기 연구는 수작업 설계 프롬프트 템플릿, 메모리 모듈, 의사결정 휴리스틱으로 에이전트 루프를 구성했다. 이런 정책 진화 접근은 자기반성·공적응·프롬프트 재작성으로 추론 과정을 정제하며 RAG 같은 추론 시점 강화 기법과 효과적으로 통합된다.

두 번째 연구 물결은 OpenAI o1, DeepSeek-R1 같은 추론 중심 모델에서 개척된 RLVR(검증자·보상 기반 RL) 패러다임을 에이전트 설정으로 확장한다. WebAgent-R1, Search-R1, R1-Searcher 같은 연구는 규칙 기반 보상 신호로 상호작용 환경에서 LLM 에이전트를 종단간 RL로 훈련해, 정적 지시 튜닝 데이터셋에만 의존하지 않고 계획·도구 사용·장기 전략을 지속 정제하게 한다.

병행하여 자기학습 대형 추론 모델(LRM) 경향이 커지고 있다 — 대량의 주석 인간 데이터 없이 RL로 추론 능력을 개선하려는 것. Absolute-Zero, R-Zero 같은 방법은 라벨 없는 데이터 조건에서 자기진화 LRM을 달성한다. 그러나 대부분 기존 접근은 RLVR 패러다임 내에서 검증 가능한 보상을 제공하기 위해 여전히 인간 주석 데이터나 규칙 기반 환경(예: 코드 인터프리터)에 크게 의존한다. 이런 의존은 자동 검증 가능한 피드백이 드물고 과제 구조가 덜 형식화된 오픈도메인으로 자기학습을 확장하는 잠재력을 크게 제약한다.

이에 동기를 얻어, 우리는 자기학습 패러다임을 에이전트 시스템 훈련 확장으로 효과적으로 확장해 인간 큐레이션 데이터와 사전 정의 규칙 기반 보상 함수 양쪽으로부터 자유롭게 할 수 있는지 탐구한다. Search-R1이 도입한 검색 에이전트 환경에서 통제 실험을 시작해 두 핵심 요인(보상 신호 출처, 에이전트 과제 데이터 규모)을 조사한다. 발견: (1) 경직된 규칙 기반 보상보다 GRM이 만든 신호가 오픈도메인 확장에 더 효과적이며, GRM을 정책과 공진화시키면 성능이 더 향상. (2) 에이전트 과제 데이터 양 증가는 — 합성이라도 — 능력을 크게 향상.

이를 바탕으로 ASL을 제시한다 — 공유 도구 환경과 LLM 백본 안에서 과제 생성·정책 실행·평가를 통합하는 완전 폐루프 다중 역할 RL 프레임워크. 세 역할(프롬프트 생성기, 정책 모델, GRM)의 반복 최적화로 자기개선 루프를 구현한다.

## 한 줄 핵심
검색 에이전트를 인간 데이터·규칙 보상 없이, 문제 출제기·풀이 정책·생성형 보상 모델 세 역할이 서로 공진화하는 폐루프로 자기학습시키는 RL 프레임워크(ASL).
