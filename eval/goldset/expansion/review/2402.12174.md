# 2402.12174 BIDER: Bridging Knowledge Inconsistency for Efficient Retrieval-Augmented LLMs via Key Supporting Evidence

> section_tag: 정보필터링  ·  survey_source: s2

## 초록 (한글)
검색 증강 대규모 언어 모델(LLM)은 오픈 도메인 QA 같은 지식 집약적 과제에서 효과를 입증했으며, 지식 갱신과 사실성 부족이라는 본질적 문제를 완화한다. 그러나 검색된 지식과 LLM이 실제로 필요로 하는 지식 사이의 불일치는 LLM 답변 품질을 떨어뜨린다. 본 논문은 검색 문서를 핵심 근거(Key Supporting Evidence, KSE)로 정제하는 접근법 BIDER를 제안한다. 이는 지식 합성, 지도 미세조정(SFT), 선호도 정렬(preference alignment)의 세 단계로 이루어진다. KSE를 만드는 법을 학습시키는 동시에, 강화학습을 통해 출력이 LLM의 정보 획득 선호도에 맞도록 최대화한다. 다섯 개 데이터셋 평가에서 BIDER는 검색 문서의 입력 길이를 80% 줄이면서도 LLM 답변 품질을 7% 향상시켜 기존 방법을 능가했다. 제안한 KSE 시뮬레이션은 정확한 질의응답에 필요한 필수 정보를 LLM에 효과적으로 공급한다.

## Introduction (한글)
LLM은 빠르게 발전하며 막대한 능력을 보이지만, 지식 갱신과 사실 기반 응답 생성, 특히 오픈 도메인 QA 같은 지식 집약적 과제에서 한계가 있다. 이를 해결하기 위해 검색 증강 생성(RAG)이 유망한 접근으로 등장했으며, 외부 지식을 통합해 LLM의 단점을 완화한다. 표준 RAG는 질문 관련 문서를 검색해 프롬프트에 직접 넣어 답을 생성하지만, 검색 시스템의 불완전성과 LLM 자기지식 접근 불가로 인해 검색 문서가 길고 노이즈가 많아 생성 품질을 해친다. 최근 연구들은 정렬 메커니즘으로 관련 문장만 남기거나, 요약하거나, 기여도가 낮은 내용을 제거하는 식으로 검색 문서를 최적화해 왔다. 그러나 이들은 생성기 피드백에 과도하게 의존하여 질문 자체에 필요한 본질적 지식을 간과하며, 이는 불안정하고 핵심 정보 손실/노이즈 잔류를 초래할 수 있다. 저자들은 이 한계가 검색 결과와 모델이 답에 진짜 필요한 지식 사이의 불일치를 무시한 데서 비롯된다고 보고, 이 본질적 지식을 핵심 근거(KSE)로 정의한다. BIDER는 (1) 지식 합성(3단계로 진짜 KSE 합성) (2) 지도 미세조정(검색 문서→KSE 매핑을 학습하는 seq2seq 모델) (3) 선호도 정렬(강화학습으로 다운스트림 LLM 선호에 정렬)의 세 단계로 검색 문서를 KSE로 정제한다. NQ, TQA, HotpotQA(오픈 도메인 QA), WoW(대화), FEVER(사실 검증) 다섯 데이터셋에서 입력 길이를 80% 줄이면서 더 나은 성능을 달성했다.

## 한 줄 핵심
검색된 문서를 LLM이 답에 진짜 필요한 "핵심 근거(KSE)"로 압축·정제하는 정제기 모델을 지도학습+강화학습(선호 정렬)으로 훈련해, 입력 길이를 80% 줄이면서 RAG 답변 품질을 높이는 방법.
