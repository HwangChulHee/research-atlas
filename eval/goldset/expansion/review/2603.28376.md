# 2603.28376 Marco DeepResearch: Unlocking Efficient Deep Research Agents via Verification-Centric Design

> section_tag: 검색시점;RAG강화추론  ·  survey_source: s3

## 초록 (한글)
딥 리서치 에이전트는 열린 형태의 조사를 자율적으로 수행하며, 복잡한 정보 검색과 다단계 추론을 다양한 소스에 걸쳐 통합해 실세계 문제를 푼다. 긴 호흡 과제에서 이 능력을 지속하려면 훈련과 추론 모두에서 신뢰할 만한 검증이 핵심이다. 기존 패러다임의 주요 병목은 QA 데이터 합성, 궤적 구성, 테스트 타임 스케일링에서 명시적 검증 메커니즘의 부재에서 비롯된다. 각 단계에서 도입된 오류가 다운스트림으로 전파되어 전반적 에이전트 성능을 저하시킨다. 이를 해결하기 위해 우리는 세 수준에서 검증 중심(verification-centric) 프레임워크로 최적화한 딥 리서치 에이전트 Marco DeepResearch를 제시한다: (1) QA 데이터 합성: 그래프 기반·에이전트 기반 QA 합성에 검증 메커니즘을 도입해 질문 난이도를 제어하면서 답이 유일·정확하도록 보장한다; (2) 궤적 구성: 훈련 궤적에 명시적 검증 패턴을 주입하는 검증 주도 궤적 합성 방법을 설계한다; (3) 테스트 타임 스케일링: 추론 시 Marco DeepResearch 자체를 검증기로 사용해 어려운 질문의 성능을 효과적으로 개선한다. 광범위한 실험 결과는 Marco DeepResearch가 BrowseComp, BrowseComp-ZH 같은 가장 어려운 벤치마크에서 8B 규모 딥 리서치 에이전트를 크게 능가함을 보인다. 결정적으로, 최대 600 도구 호출 예산 하에서 Marco DeepResearch는 Tongyi DeepResearch-30B 같은 여러 30B 규모 에이전트를 능가하거나 근접한다.

## Introduction (한글)
LLM은 자율적으로 추론하고 외부 환경과 상호작용하는 도구 증강 에이전트를 가능하게 했다. 이 흐름에서 딥 리서치 에이전트는 큰 주목을 받았다: OpenAI Deep Research, Gemini Deep Research 같은 상용 시스템은 복잡한 실세계 과제 해결에서 뛰어난 정보 탐색 능력을 보이고, MiroThinker, Tongyi DeepResearch, AgentCPM-Explore 같은 오픈소스 시스템은 격차를 빠르게 좁혀 일부 설정에서는 상용을 따라잡거나 능가했다. 이 진전은 주로 데이터 합성, 강화학습, 테스트 타임 스케일링의 발전에 의해 추동되었다.

이 진전에도 현재 딥 리서치 에이전트는 결정적 병목에 직면한다: 다음 세 핵심 단계에서 명시적 검증의 부재로 오류 전파와 전반적 성능 저하가 발생한다. (1) QA 데이터 합성: 대부분의 연구가 그래프 기반 또는 에이전트 기반 웹 탐색에서 QA 샘플을 합성하나, 기존 파이프라인에서 가장 널리 쓰이는 기법인 엔티티 난독화(entity obfuscation)는 종종 유일하지 않거나 부정확한 답을 낳아 감독 품질을 훼손하고 다운스트림 궤적 구성으로 오류를 전파한다. (2) 궤적 구성: 대부분의 기존 연구는 강한 교사 모델로 정답에 직접 도달하는 ReAct 스타일 궤적을 생성하지만, 이러한 궤적은 보통 명시적 검증이 없어, 훈련된 에이전트가 이른 저품질 결과를 수용하고 고가치 대안을 충분히 탐색하지 못하는 경향을 보인다. (3) 테스트 타임 스케일링: 시스템은 일반적으로 추론 시 중간 단계와 최종 답 모두에 대한 명시적 검증이 없어, 결함 있는 중간 상태와 잘못된 결론이 점검 없이 전파되어 검증기 유도 행동으로 테스트 타임 연산을 효과적으로 스케일하기보다 이른 오류를 수용하게 된다.

이러한 검증 간극을 해결하기 위해 우리는 세 단계에 걸친 세 가지 개선을 갖춘 효율적 8B 규모 딥 리서치 에이전트 Marco DeepResearch를 제시한다: (1) 검증된 데이터 합성(3절): 그래프 기반·에이전트 기반 QA 합성에 명시적 검증 메커니즘을 도입해 질문-답 쌍의 난이도·유일성·정확성을 면밀히 점검한다; (2) 검증 주도 궤적 구성(4절): 전문 검증기 에이전트를 도입해 웹 검색 도구로 하위 과제 답과 최종 답을 검증하여 단일·다중 에이전트 궤적에 더 명시적인 검증 패턴을 주입한다; (3) 검증기 유도 테스트 타임 스케일링(5절): Marco DeepResearch 에이전트 자체를 검증기로 사용해 통제된 연산 예산 하에 어려운 질문에 대한 추론을 계속함으로써 테스트 타임 스케일링의 잠재력을 더 효과적으로 끌어낸다.

이 최적화로 Qwen3-8B 기반 모델 위에 고품질 궤적을 합성·훈련해 BrowseComp, BrowseComp-ZH, GAIA 등 여섯 딥서치 벤치마크에서 평가한다. Marco DeepResearch는 가장 어려운 벤치마크에서 8B 규모 에이전트를 능가하고, 최대 600 도구 호출 예산 하에서 MiroThinker-v1.0-8B를 능가하며 Tongyi DeepResearch-30B, MiroThinker-v1.0-30B 같은 여러 30B 규모 에이전트를 따라잡거나 넘어선다. 절제 연구가 설계들의 기여를 입증한다.

## 한 줄 핵심
딥 리서치 에이전트의 QA 데이터 합성·궤적 구성·테스트 타임 스케일링 세 단계 모두에 명시적 검증을 심어 오류 전파를 막고, 8B로 30B급 성능에 도달한 검증 중심 에이전트.
