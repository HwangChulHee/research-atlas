# 2605.29307 GrepSeek: Training Search Agents for Direct Corpus Interaction

## 초록 (한글)
LLM 검색 에이전트는 여러 라운드의 추론과 정보 검색을 통해 지식 집약적 언어 과제에 강한 잠재력을 보였다. 대부분의 기존 시스템은 키워드나 자연어 질의를 받아 사전 계산된 문서 표현 인덱스로 순위화된 문서 목록을 반환하는 검색기를 통해 정보에 접근한다. 본 연구는 검색 에이전트가 코퍼스 자체를 검색 환경으로 취급하고 실행 가능한 셸 명령을 발행해 증거를 찾는 보완적 관점을 탐구한다. 대형 텍스트 코퍼스에서 증거를 찾고 거르고 구성하도록 컴팩트 검색 에이전트를 훈련하는 최적화된 직접 코퍼스 상호작용(DCI) 검색 에이전트 GrepSeek을 도입한다. 대형 코퍼스에서 RL로 직접 행동을 학습하는 불안정성을 다루기 위해 2단계 훈련 파이프라인을 제안한다. 첫째, 정답 인지 Tutor와 정답 무지 Planner로 검증되고 인과적으로 근거화된 검색 궤적을 생성해 콜드스타트 데이터셋을 구축한다. 둘째, GRPO로 초기화된 정책을 정제해 에이전트가 코퍼스와의 직접 상호작용으로 과제 지향 검색 행동을 개선하게 한다. DCI를 대규모에서 실용화하기 위해, 셸 기반 검색을 최대 7.6배 가속하면서 순차 실행과 바이트 단위 동등성을 보존하는 의미 보존 샤드 병렬 실행 엔진을 사용한다. 일곱 개 오픈도메인 QA 벤치마크 실험에서 GrepSeek은 토큰 수준 F1과 정확 일치에서 최강의 전체 성능을 달성한다. 분석은 표면형 변이가 큰 질의에서 순수 어휘 상호작용의 한계도 보여, DCI를 기존 검색 패러다임을 보완하는 실용적·경쟁력 있는 방법으로 제시한다.

## Introduction (한글)
LLM 검색 에이전트는 추론, 질의 분해, 다수 출처 정보 종합을 요하는 복잡한 정보 요구 처리에 강한 잠재력을 보였다. 이들은 지식 집약적 과제 수행에 필요한 정보를 얻기 위해 검색 모델과 여러 번 상호작용한다. 비구조·반구조 텍스트 코퍼스에서 이 상호작용은 키워드나 자연어 질의 형태다. 이런 접근은 어휘 매칭(Salton & Buckley 등)부터 밀집·희소 표현 기반 의미 매칭에 이르는 수십 년 검색 연구에 의존한다. 이 모델들은 문서의 사전 계산 표현으로 코퍼스 인덱스를 구성하고 문서별 관련성 점수를 계산한다.

본 논문은 비구조 데이터의 정보 탐색을 임의 입도에서 할 수 있다는 새로운 관점을 탐구한다. 즉 문서 수준 표현·인덱싱·관련성 점수 대신, 임의 크기의 텍스트 조각을 질의마다 검색할 수 있게 해, 사전 결정된 청크·표현에 제한되지 않는 더 "수술적" 검색을 수행한다. 이를 위해 코퍼스를 환경으로 취급하는 검색 에이전트를 구상한다—에이전트는 실행 가능한 (수술적) 검색 연산을 코퍼스에 발행하고, 중간 결과를 검사하며, 제약을 정제하고, 여러 단계에 걸쳐 증거를 구성한다. 이는 검색을 블랙박스 순위화 절차에서 제어 가능한 코퍼스 연산의 명시적 시퀀스로 전환한다. 이런 인터페이스는 정확한 개체 매칭, 어휘 필터링, 기호적 패턴 검색, 문서 간 브리지 개체 추적이 필요한 지식 집약적 추론 과제에 특히 매력적이다.

이 관점은 grep, ripgrep 같은 실행 가능 검색 도구가 코드 저장소에서 관련 맥락을 찾는 단순하지만 효과적인 인터페이스를 제공하는 최근 코드 에이전트 진전과 밀접하다. 이 도구 매개 검색 형태에서 영감을 받아, 유사한 상호작용 패턴을 코드 저장소를 넘어 수백만 비구조 문서의 대형 텍스트 코퍼스 오픈도메인 QA로 확장할 수 있는지 묻는다.

## 한 줄 핵심
검색기 인덱스 대신 코퍼스를 환경으로 보고 셸 명령(grep류)을 발행해 증거를 찾는 "직접 코퍼스 상호작용" 검색 에이전트를, 콜드스타트(Tutor/Planner) + GRPO 2단계로 훈련한다.
