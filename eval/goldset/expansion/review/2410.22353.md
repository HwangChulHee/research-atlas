# 2410.22353 RuleRAG: Rule-Guided Retrieval-Augmented Generation with Language Models for Question Answering

> section_tag: 질의계획  ·  survey_source: s1

## 초록 (한글)
검색증강생성(RAG)은 지식집약적 질의응답(QA)에서 유망한 잠재력을 보여왔다. 그러나 기존 접근법은 질의 자체만 고려할 뿐, 검색기에 대한 검색 선호도를 명시하지도 않고, 생성기에게 검색된 문서를 답변에 어떻게 참조할지 알려주지도 않아 QA 성능에 큰 도전이 된다. 이를 해결하기 위해 우리는 LM을 이용한 규칙 기반 검색증강생성(RuleRAG)을 제안한다. 이는 인컨텍스트 학습을 위한 규칙(RuleRAG-ICL)을 명시적으로 도입하여, 검색기가 규칙 방향으로 관련 문서를 회수하도록 안내하고 생성기가 동일 규칙에 귀속하여 추론하도록 일관되게 안내한다. 질의와 규칙의 결합은 검색기와 생성기를 갱신하는 파인튜닝 데이터로 사용되어, 규칙 기반 지시 따르기 능력을 향상시킨다(RuleRAG-FT). 또한 대부분의 기존 RAG 데이터셋은 규칙을 고려하지 않고 구축되었으며, 지식그래프(KG)는 고품질 규칙을 제공하는 것으로 알려져 있다. 따라서 우리는 KG를 기반으로 규칙 인식 RAG 벤치마크 5개(RuleQA)를 구축하여 규칙 기반 검색·추론의 중요성을 강조한다. RuleQA 실험에서 RuleRAG-ICL은 Recall@10에서 +89.2%, Exact Match에서 +103.1%의 향상을 보였고, RuleRAG-FT는 더 큰 향상을 낳았다. 또한 기존 RAG 데이터셋 4개 실험에서도 RuleQA의 규칙을 적용하면 효과적임을 보여, 규칙 안내의 일반화를 입증한다.

## Introduction (한글)
대형언어모델(LLM)은 인상적인 언어 생성·지식 학습 능력을 달성했으나, 전(全)파라미터 지식으로는 특히 지식집약적 과제에서 세밀한 질의를 정밀하게 다루기 어렵다. 이를 보완하여 RAG는 오픈도메인 QA, 자연어추론 등 많은 NLP 과제에서 우수한 성능을 보인다.

그러나 현재 RAG에는 두 가지 고차원 문제가 있다. 첫째, 검색 단계에서 검색기는 단어 수준 매칭에 의존하므로 회수된 정보가 항상 질의 답변에 적절하다고 보장할 수 없다. 둘째, 생성 단계에서 현재 RAG의 LLM은 잡음 섞인 검색 내용을 어떻게 적절히 활용할지 명시적으로 안내받지 못한다. 사실 간 관계가 사전학습 코퍼스에서 명시적으로 "지적"되거나 "감독"되는 경우가 드물기 때문이다. 따라서 현재 RAG는 합리적 검색 방향으로 검색하도록 훈련되지도, 검색 내용을 답변에 유기적으로 귀속시키지도 못한다.

단순 단어 매칭 대신 사전 규칙(a priori rules)은 복잡한 지식 사이의 내부 논리 패턴을 진정으로 포착할 수 있다. 일부 연구는 규칙을 LLM에 통합해 숫자 덧셈이나 산업 과제를 다루었으나, RAG QA에 규칙을 도입한 탐구는 아직 없다. 예시(Figure 1)에서 "YSSTECH 주가의 향후 추세는?"이라는 질의에 대해, 금융 KG는 "회사 사업의 합병이 주가 상승으로 이어진다"는 규칙을 제공하며 이를 활용해 더 표적화된 검색과 답변 지원이 가능하다.

이러한 동기에서 우리는 RuleRAG를 제안한다. 학습 불필요한 RuleRAG-ICL은 검색기와 생성기 입력 측에 규칙을 도입하고, 규칙 따르기 추론 능력을 높이기 위해 규칙 기반 파인튜닝(RGFT)을 추가 설계한다(RuleRAG-FT). 또한 KG로부터 규칙 인식 QA 벤치마크 5개(RuleQA)를 새로 구축한다. 실험 결과 RuleRAG-ICL과 RuleRAG-FT 모두 상당한 향상을 보이며, RuleRAG-FT는 재학습 없이 미지의 규칙으로 외삽 가능하다. 우리는 또한 고급 모델 CoK에 RuleRAG를 도입하여(RuleRAG-CoK) 규칙 기반 RAG 패러다임이 다양한 RAG 방법에 적합함을 강조한다.

## 한 줄 핵심
규칙(KG에서 추출한 논리 규칙)을 검색기와 생성기 양쪽에 인컨텍스트 또는 파인튜닝으로 주입하여, 더 표적화된 검색과 귀속 가능한 추론을 이끄는 RAG 방법이다.
