# 2508.16438 OPERA: A Reinforcement Learning-Enhanced Orchestrated Planner-Executor Architecture for Reasoning-Oriented Multi-Hop Retrieval

> section_tag: 질의계획;에이전트오케스트레이션;학습형RL  ·  survey_source: s2;s3

## 초록 (한글)
대형 언어 모델(LLM)과 밀집 검색기(dense retriever)의 최근 진전은 검색 증강 생성(RAG)에서 큰 발전을 이끌었다. 그러나 기존 접근은 복잡한 추론 지향 멀티홉 검색 과제에서 중대한 난제에 직면한다. 1) 비효과적 추론 지향 계획: 규칙 기반 분해기는 템플릿 밖 질문에서 성능이 나빠 복잡 질의에 대한 견고한 다단계 계획을 생성하기 어렵다. 2) 부적절한 추론 주도 검색: 관련 방법은 제한된 질의 재구성을 사용해 골든 문서를 찾지 못하는 반복 검색 루프에 빠진다. 3) 불충분한 추론 유도 필터링: 기존 방법은 잡음 결과에서 핵심 정보를 거르는 세밀한 추론이 부족하다. 근본적으로 이 한계들은 모두 현재 RAG 아키텍처에서 검색과 추론 사이의 약한 결합(weak coupling)에서 비롯된다. 본 논문은 새로운 추론 주도 검색 프레임워크 OPERA(Orchestrated Planner-Executor Reasoning Architecture)를 도입한다. OPERA의 목표 계획 모듈(GPM)은 질문을 하위 목표로 분해하고, 이를 정밀한 추론과 효과적 검색을 위한 전용 구성 요소를 갖춘 추론-실행 모듈(REM)이 실행한다. OPERA 학습을 위해 GRPO의 새 변형인 Multi-Agents Progressive GRPO(MAPGRPO)를 제안한다. 복잡한 멀티홉 벤치마크 실험은 OPERA의 우수한 성능을 보여 MAPGRPO 방법과 OPERA 설계 모두를 검증한다.

## Introduction (한글)
복잡한 문제 해결은 지능의 핵심이며, RAG 안에서 추론 중심 검색은 사후학습 시대에 이를 다루는 효과적 수단을 제공한다. LLM과 밀집 검색 시스템의 동시 향상은 RAG 진화를 추동했다. 전통적 RAG는 retrieve-then-reason 패러다임을 따르며, 이제 질의 재작성·문서 검색·문서 필터링·답 생성을 포함하는 다단계 파이프라인으로 널리 최적화되었다. 그러나 이 능력들을 효과적으로 오케스트레이션하는 것은 여전히 어렵고, 기존 접근은 멀티홉 추론 요구에 고전한다. 예: "WhatsApp을 인수한 회사의 창립자를 승계한 사람의 이전 직업은?" 같은 질문은 단순 문서 검색이 아니라 각 연산이 선행 연산에 결정적으로 의존하는 정밀한 검색·추론 시퀀스의 오케스트레이션을 요구한다.

현재 접근의 핵심 난제. 첫째, 추론 지향 계획이 제한적이다. PlanRAG, REAPER 같은 planner-first 모델은 선행 계획 메커니즘을 도입하지만, 정적 계획이 검색 중 예기치 못한 도전에 동적으로 적응하지 못한다. 둘째, 추론 인식 검색이 부족하다. Adaptive-RAG, AT-RAG처럼 질의 복잡도에 따라 검색 전략을 조정하는 적응형 방법조차 복잡 멀티홉에 필요한 세밀한 추론 주도 질의 재구성이 없다. 셋째, 추론 유도 필터링이 미흡하다. 검색이 골든 문서를 가져와도 잡음 많은 top-K에 묻히는데, ReAct는 추론-행동 루프로, BGM은 검색기-LLM 선호 연결로 이를 다루려 하지만 간접 보상 신호 탓에 효과가 제한된다. 이 한계들은 SFT·선호 최적화·RL 같은 향상 전략이 불충분하거나 정렬되지 않아 모듈 간 목표 불일치를 낳기 때문에 지속되며, 근본적으로 검색과 추론 사이 결합의 약함에서 비롯된다.

이를 해결하기 위해 추론 주도 프레임워크 OPERA를 도입한다. OPERA는 고수준 전략 계획과 저수준 전술 실행을 두 핵심 모듈로 체계적으로 분리한다: 목표 계획 모듈(GPM)과 추론-실행 모듈(REM). GPM은 전용 Plan Agent로 복잡 질의를 일관·실행 가능한 하위 목표로 분해한다. REM은 밀집 검색기를 지원받는 이중 에이전트 시스템으로, Analysis-Answer Agent가 검색 맥락에서 정밀한 답을 추출하고 전문 Rewrite Agent가 후속 검색 개선을 위해 질의를 재구성한다. 또한 OPERA는 각 행동의 근거를 명확히 제공해 해석 가능성을 높이는 궤적 메모리 구성 요소(TMC)를 갖는다. 학습 프로토콜은 세 에이전트 각각을 역할 맞춤 GRPO 보상(계획 품질, 추론 정확도, 검색 효과)으로 순차 최적화한다. 핵심 기여: (1) 추론 주도 검색 프레임워크(TMC 포함), (2) 세밀한 역할별 신용 할당을 하는 전문 학습 알고리즘 MAPGRPO, (3) 멀티홉 벤치마크에서의 강한 실증 결과.

## 한 줄 핵심
멀티홉 RAG에서 계획(GPM)과 실행(REM: 분석-답변 + 재작성 에이전트)을 분리한 플래너-실행자 아키텍처를 만들고, 역할별로 순차 최적화하는 MAPGRPO(GRPO 변형)로 학습해 검색-추론 결합을 강화한 방법.
