# 2511.01854 Tool-to-Agent Retrieval: Bridging Tools and Agents for Scalable LLM Multi-Agent Systems

> section_tag: RAG강화추론  ·  survey_source: s1

## 초록 (한글)
LLM 다중 에이전트 시스템의 최근 진전은 하위 에이전트의 확장 가능한 오케스트레이션을 가능하게 하며, 각 하위 에이전트는 수백~수천 개의 도구나 모델 컨텍스트 프로토콜(MCP) 서버를 조율한다. 그러나 기존 검색 방법은 보통 질의를 거친 에이전트 수준 설명에 매칭한 뒤 라우팅하는데, 이는 세밀한 도구 기능을 가려 차선의 에이전트 선택으로 이어진다. 우리는 Tool-to-Agent Retrieval을 소개한다. 이는 도구와 그 부모 에이전트를 공유 벡터 공간에 임베딩하고 메타데이터 관계로 연결하는 통합 프레임워크다. 도구 능력을 명시적으로 표현하고 메타데이터를 에이전트 수준까지 순회함으로써, Tool-to-Agent Retrieval은 세밀한 도구 수준 또는 에이전트 수준 검색을 가능하게 하여, 많은 도구를 하나로 묶을 때 생기는 문맥 희석(context dilution) 없이 에이전트와 그 하위 도구·MCP 서버가 동등하게 표현되도록 보장한다. 8개 임베딩 모델에 걸쳐 평가한 결과, 우리 접근은 LiveMCPBench 벤치마크에서 이전 최첨단 에이전트 검색기 대비 Recall@5에서 19.4%, nDCG@5에서 17.7%의 일관된 향상을 달성했다.

## Introduction (한글)
LLM 에이전트와 MCP의 최근 발전으로, 어시스턴트는 추론 시점에 대규모 외부 도구·MCP 서버 모음을 발견·장착·사용할 수 있다. 실제로 단일 어시스턴트는 코드 분석·데이터베이스·웹 검색을 위한 전문 하위 에이전트에 위임하며, 각 에이전트는 단일 인터페이스 뒤에 수십 개 도구를 묶는다. 핵심 도전은 라우팅이다 — 사용자 질의가 주어졌을 때 특정 도구를 선택할지, 아니면 일관된 도구 집합을 제공하는 전체 에이전트(예: MCP 서버)를 활용할지. 모든 도구 설명을 모델에 전달하는 것은 비현실적이다(예: 26개 도구를 가진 한 MCP 서버는 4,600토큰 이상 소비). 따라서 효율적 검색이 확장성에 필수다.

기존 전략은 보통 두 진영으로 나뉜다. 에이전트 우선(Agent-first) 파이프라인은 질의를 간략한 에이전트 설명에 매칭한 뒤 그 에이전트 안에서만 작동하는데, 부모 설명이 질의와 명확히 맞지 않는 매우 관련성 높은 도구를 숨길 수 있다. 반대로 도구 전용(tool-only) 검색은 각 도구를 독립적으로 다뤄, 다단계 과제에서 주변 번들의 보완적 이점을 무시한다. 단일 도구 매칭은 워크플로우가 조율된 도구 집합에서 이득을 볼 때 성능이 떨어질 수 있고, 에이전트 우선 라우팅은 세밀한 능력을 놓칠 수 있다. 필요한 것은 양 수준에서 작동할 수 있는 검색 메커니즘 — 질의가 구체적이면 집중된 도구를, 더 넓은 능력이 유리하면 전체 에이전트를 반환하되, 취약한 2단계 파이프라인에 얽매이지 않는 것.

우리는 Tool-to-Agent Retrieval을 소개한다. 이는 도구와 그 부모 에이전트를 같은 벡터 공간에 표현하고 명시적 메타데이터로 연결한다. 질의 시 결합 인덱스 위에서 검색이 실행되어 질의에 더 맞는 개별 도구 또는 에이전트 번들을 반환한다. 세밀한 도구 의미를 모델링하면서 도구→에이전트 링크로 에이전트 문맥을 유지(검색 시 이 링크를 순회)함으로써, 많은 도구를 단일 거친 설명으로 뭉칠 때 생기는 문맥 희석을 피한다.

(관련 연구 — 압축) 도구 설명·메타데이터를 도구 지식베이스에 임베딩해 질의당 top-k 도구를 선택하는 연구 계열; RAG로 LLM 에이전트가 의미 검색으로 외부 지식·도구에 접근; 지식 그래프와 벡터 검색을 결합한 하이브리드; PLUTO·Re-Invoke(밀집 검색 정제); ToolNet·ControlLLM(그래프 기반 도구 관계 구조화); ToolReAGT(반복적 단계별 도구 검색) 등.

## 한 줄 핵심
도구와 그 부모 에이전트를 하나의 벡터 공간에 함께 임베딩하고 메타데이터 링크로 잇어, 질의에 따라 개별 도구 또는 에이전트 번들을 한 번에 검색하는 다중 에이전트 라우팅 프레임워크.
