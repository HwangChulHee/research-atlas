# 2504.04915 Collab-RAG: Boosting Retrieval-Augmented Generation for Complex Question Answering via White-Box and Black-Box LLM Collaboration

> section_tag: 질의계획;에이전트오케스트레이션  ·  survey_source: s1

## 초록 (한글)
검색 증강 생성(RAG) 시스템은 무관한 맥락 검색과 제한된 복잡 추론 능력 때문에 다중 홉 QA 과제를 정확히 처리하는 데 종종 어려움을 겪는다. 본 연구는 화이트박스 소형 언어모델(SLM)과 블랙박스 대형 언어모델(LLM) 간의 상호 강화를 활용하는 협력 학습 프레임워크 Collab-RAG를 소개한다. 구체적으로, SLM이 복잡한 질의를 더 단순한 하위 질문들로 분해하여 검색 정확도를 높이고 블랙박스 LLM의 더 효과적인 추론을 촉진한다. 동시에 블랙박스 LLM은 SLM의 분해 능력을 개선하기 위한 피드백 신호를 제공한다. Collab-RAG는 최첨단 LLM으로부터의 추가 증류(distillation) 없이 저렴한 블랙박스 LLM의 감독만으로 학습되지만, 여러 블랙박스 LLM에 걸쳐 강한 일반화를 보인다. 5개 다중 홉 QA 데이터셋 실험에서 Collab-RAG는 기존 블랙박스 전용 및 SLM 미세조정 기준선을 평균 1.8%~14.2% 능가했다. 특히 미세조정된 3B SLM이 질문 분해에서 동결된 32B LLM을 능가하여 효율성을 입증했다.

## Introduction (한글)
LLM은 강력하지만 환각, 진화하거나 도메인 특화된 지식에의 적응 어려움 등의 한계가 있다. RAG는 외부 지식을 통합해 사실 정확도와 신뢰성을 높여 이를 완화한다. 표준 RAG 파이프라인에서는 검색 모델이 질의에 기반해 관련 정보를 찾고, 검색된 맥락이 질의와 결합되어 LLM에 입력된다. 실무에서는 더 강한 블랙박스 LLM을 RAG 리더로 흔히 쓰지만, 검색 단계에서 무관한 맥락이 들어오면 LLM을 오도해 성능이 만족스럽지 못한 경우가 많다. 이 문제는 여러 증거가 필요한 복잡 QA에서 특히 두드러진다.

복잡 질문 처리를 위해 여러 연구가 검색 품질 개선을 시도했다. 보조 임베딩 모델 학습이나 질의 재순위화 등이 있으나, 이들은 단일 스텝 검색 개선에 집중해 반복적 증거 수집이 필수인 복잡 QA의 본질적 어려움을 다루지 못한다. 또 일부 연구는 LLM 자체로 질의를 정제하거나 다중 턴 검색을 수행하는 학습 불요(training-free) 방법을 제안했으나, 전용 학습 없이는 효과적 질의 분해·정제 능력이 제한적이다. 최근 소형 언어모델을 미세조정해 RAG 성능을 높이는 연구도 있으나, 블랙박스 LLM의 파라미터 갱신은 비효율적·자원 집약적이라 실용성이 제한된다.

본 연구는 화이트박스 SLM을 추가로 결합해 복잡 QA에서 블랙박스 LLM을 강화하는 Collab-RAG를 제안한다. SLM은 분해기(decomposer)로서 복잡 질의를 더 작은 원자적 하위 질문으로 쪼개 관련 맥락 검색을 개선하고, 블랙박스 LLM은 리더로서 각 하위 질문의 중간 답을 생성한 뒤 최종 응답으로 종합한다. SLM을 그대로 질문 분해에 쓰는 것은 추론 능력 한계로 비효과적이고 대규모 고품질 주석 수집은 비싸므로, 블랙박스 LLM(GPT-4o-mini)의 피드백만 사용하는 자기 개선 학습 전략을 제안한다. 블랙박스 LLM을 응답·질의·검색맥락을 생성하는 환경으로 모델링하고, SLM이 이 환경과 다중 턴 상호작용하며 분해 전략을 반복 정제한다. 핵심 통찰은 "더 좋은 질문 분해가 더 좋은 최종 답으로 이어진다"는 것이며, 이에 기반해 블랙박스 LLM 피드백을 쓰는 반복적 선호 최적화(iterative preference optimization) 접근을 설계한다. 또한 규칙 기반 평가로 하위 질문의 형식과 최종 답 정확도를 평가해 효과적/비효과적 분해를 구분한다. (이하 기여 압축)

## 한 줄 핵심
작은 화이트박스 모델이 복잡 질문을 하위 질문으로 분해하고, 블랙박스 LLM의 답 정확도 피드백으로 그 분해 능력을 선호 최적화로 학습시켜 다중 홉 RAG를 개선하는 협력 프레임워크.
