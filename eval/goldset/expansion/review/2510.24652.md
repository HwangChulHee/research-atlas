# 2510.24652 R3: Optimizing Retrieval for RAG via Reinforcement Learning

> section_tag: 질의계획  ·  survey_source: s1

## 초록 (한글)
검색 증강 생성(RAG)이 보편화되면서, 검색의 역할은 인간 열람을 위한 정보 검색에서 AI 추론을 위한 문맥 검색으로 이동하고 있다. 이 전환은 관련성(relevance)을 사전 정의하기 어려운 더 복잡한 검색 환경을 만든다. 기존 검색기(retriever)는 인간 라벨이나 합성 데이터를 사용한 지도 미세조정(SFT)에 의존하여, 다양한 RAG 환경에 적응하기 어려운 정적(static) 관련성을 낳는다. 이를 해결하기 위해 우리는 강화학습(RL)을 통해 RAG에 최적화된 검색 프레임워크 R3를 제안한다. 구체적으로, 검색기가 주어진 RAG 환경 안에서 탐색하고 자기개선하도록 하는 RL 훈련 패러다임을 채택하여, 수작업 실험이나 튜닝 노력을 최소화하면서 학습 과정을 자동화한다. 다양한 과제에 대한 광범위한 실험에서 R3는 원본 검색기 대비 RAG 성능을 5.2%, 최첨단 검색기 대비 4.9% 향상시켰으며, 사후 훈련되거나 지시 튜닝된 LLM 기반 RAG 시스템 및 LLM 증강 검색에 필적하는 결과를 달성했다. 또한 GPU 4장만으로 하루 안에 훈련을 완료해 효율적이고 실용적이다.

## Introduction (한글)
RAG는 LLM과 정보 검색(IR)을 통합해 외부 지식에 접근하는 잘 정립된 패러다임으로, 최근 새로운 고급 AI 응용의 기반으로 빠르게 진화했다 — 라이브 웹과 상호작용하는 웹 브라우징 에이전트, 검색을 통합한 대화 챗봇, 다중 출처를 자율 분석·종합해 종합 보고서를 만드는 DeepResearch 어시스턴트 등.

이런 응용은 검색 역할의 근본적 전환을 의미한다 — "인간을 위한 검색"에서 "AI를 위한 검색"으로. 전통 IR은 인간 사용자를 위해 정보를 검색하며, 관련성은 해석가능성을 위해 표면적 유사성으로 사전 정의된다. 반면 RAG는 AI 시스템을 위한 문맥적 지식을 검색하는데, AI는 방대한 정보를 효율적으로 처리·추론하고 다양·진화하는 과제를 다루도록 설계된다. 검색기는 과제·워크플로우·LLM·기타 구성요소를 아우르는 RAG 환경 안의 내부 컴포넌트로 기능하므로, 관련성이 더 환경 의존적이고 사전 정의하기 어려우며 탐색이 필요하다.

기존 RAG 응용의 검색 옵션: 검색 엔진 API는 편리하나 비싸고 확장·추가 최적화가 어렵다. 신경 검색기를 로컬 배포할 수도 있으나, 기성 검색기를 바로 쓰면 IR과 RAG 간 간극을 겪는다. 연구들은 전통 IR이 의미적으로 관련되나 도움 안 되는 콘텐츠, LLM 생성 문서, 오도하는 증거를 자주 검색해 RAG 성능을 떨어뜨린다고 보였다. Cuconasu 등은 IR이 정답 포함 문서를 검색해도 그것이 RAG 성능을 반드시 개선하진 않음을 입증했다. (LLM 충실성 연구에서도 CoT 프롬프트가 실제 추론을 반영하지 못하는 사례.) 이는 문맥 공학의 핵심 도전을 드러낸다 — 환경마다 효과적 문맥이 무엇인지 명확한 지침이 없어 수작업에 크게 의존.

추가로 부록 D 실험에서 다양한 검색기(비지도·지도·SOTA)를 IR과 RAG 설정에서 평가해 두 발견을 얻었다. Finding 1: 더 나은 IR 성능이 항상 더 나은 RAG 성능으로 이어지진 않음(특히 과제 전환 시). Finding 2: RAG 내 검색에 개선 여지가 크다 — NQ에서 77% 질의가 적어도 한 검색기로 답될 수 있으나 최고 단일 검색기는 43%만 커버.

IR에서 RAG로의 간극을 메우기 위해 R3를 제안한다. 사전 정의 관련성을 SFT로 학습하는 선행과 달리, R3는 RAG 환경 안에서 RL로 관련성을 탐색한다. 표준 RAG 워크플로우를 따라 문서를 검색·문맥화해 생성하고, 생성 결과를 바탕으로 검색된 문서를 질의 대비 긍정/부정으로 라벨링해 대조 학습으로 검색기를 최적화한다 — 이를 "강화 대조 학습(reinforced contrastive learning)"이라 부른다.

## 한 줄 핵심
"인간용 검색"이 아닌 "AI 추론용 검색"에 맞게, 검색기를 RAG 환경 안에서 생성 결과 기반 보상으로 RL 최적화(강화 대조 학습)하는 검색 프레임워크.
