# 2511.05385 TeaRAG: A Token-Efficient Agentic Retrieval-Augmented Generation Framework

> section_tag: 학습형RL  ·  survey_source: s3

## 초록 (한글)
검색 증강 생성(RAG)은 외부 지식을 활용해 대형 언어 모델(LLM)의 신뢰성을 높인다. 유연성을 위해 에이전트형 RAG(agentic RAG)는 자율적 다중 라운드 검색과 추론으로 질의를 해결한다. 최근 에이전트형 RAG가 강화학습으로 개선되었지만, 검색과 추론에서 상당한 토큰 오버헤드가 자주 발생한다. 이 트레이드오프는 효율보다 정확도를 우선한다. 이를 해결하기 위해 본 연구는 검색 콘텐츠와 추론 스텝을 모두 압축할 수 있는 토큰 효율적 에이전트형 RAG 프레임워크 TeaRAG를 제안한다. 1) 첫째, 검색된 콘텐츠는 청크 기반 의미 검색을 간결한 트리플(triplet)을 사용한 그래프 검색으로 보강하여 압축한다. 이어 의미 유사성과 공기(co-occurrence)로부터 지식 연관 그래프(knowledge association graph)를 구축하고, 개인화 PageRank(Personalized PageRank)로 그래프 내 핵심 지식을 부각해 검색당 토큰 수를 줄인다. 2) 또한 추론 스텝을 줄이기 위해 반복적 과정 인식 직접 선호 최적화(IP-DPO)를 제안한다. 보상 함수는 지식 매칭 메커니즘으로 지식 충분성을 평가하면서 과도한 추론 스텝에 페널티를 준다. 이 설계는 고품질 선호 쌍 데이터셋을 생성해 반복 DPO로 추론 간결성을 개선한다. 6개 데이터셋에서 TeaRAG는 Llama3-8B-Instruct와 Qwen2.5-14B-Instruct에서 각각 평균 Exact Match를 4%, 2% 향상시키면서 출력 토큰을 61%, 59% 줄였다.

## Introduction (한글)
LLM은 대규모 사전학습으로 큰 진전을 이뤘으나, 사전학습 중 통계적 부정확성이 환각(hallucination)을 낳을 수 있다. RAG는 외부 검색 정보를 결합해 환각을 완화하는 효과적 기법이다. RAG 정확도를 높이기 위해 전통 RAG는 보통 계획·질의 재작성·검색·재순위·정제·생성으로 구성된 사전 정의 다단계 워크플로우를 쓴다. 그러나 고정 워크플로우는 다단계 추론·적응적 의사결정·검색 정보의 동적 통합이 필요한 과제 처리 능력을 제약한다. 유연성·적응성 강화를 위해 에이전트형 RAG가 제안되었다 — LLM이 워크플로우 과정을 자율 제어해 복잡한 문제를 풀고, 과제를 다루기 쉬운 스텝 시퀀스로 분해하며, 각 스텝에서 능동적으로 검색 모듈을 호출해 추론과 검색을 통합한다.

최근 에이전트형 RAG는 RL 최적화로 크게 향상되었다. 예: Search-R1은 PPO·GRPO로 출력 기반 보상으로 에이전트형 RAG를 최적화한다. 그러나 현 패러다임은 여러 도전을 안고 있다. 첫째, 기존 에이전트형 RAG는 토큰 비효율적이다 — 현 시스템은 최종 출력 정확도 최대화에 집중하면서 추론·검색 과정의 상당한 토큰 오버헤드를 간과한다. 이는 과도사고(overthink)와 중복 검색을 유발해 자원 낭비·효율 저하를 부른다. 둘째, 현 시스템은 대체로 청크 기반 의미 검색에 의존해 정보 밀도가 낮고 무관한 잡음을 들인다. 고밀도 지식 그래프 검색 연구가 있으나 대규모 그래프에서 미검증이고 청크·트리플 간 공기 관계를 활용하지 못해 두 검색법의 강점을 결합하지 못한다. 셋째, 대부분 출력 기반 보상으로 RL 훈련하는데 보상이 희소·잡음이 많아 안정적·효율적 훈련을 저해한다. PPO·GRPO는 다중 머신 조율과 훈련 중 지속적 외부 검색 접근을 요구해 훈련이 매우 느리다. 최근 일부는 GPT-4o로 과정 기반 보상 주석 후 DPO로 최적화하지만, 주석 비용이 크고 독점 모델에 의존한다.

토큰 사용을 깊이 보기 위해 Search-R1과 R1-Searcher 두 대표 방법을 분석했다. 에이전트형 RAG 출력은 (1) LLM 사고 과정 토큰과 (2) 검색된 콘텐츠 토큰으로 구성된다. 두 핵심 비효율 식별: 검색 콘텐츠가 전체 출력의 대부분을 차지(청크 검색이 문서 세그먼트 전체를 반환, 상당수는 무관·중복 배경) → 검색 콘텐츠 정보 밀도 향상이 필수. 그리고 단일 홉 질문에도 다단계 추론을 함(테스트셋 44%가 단일 홉인데도 대부분 1스텝 초과) → 중간 과정의 불필요·반복 스텝 축소가 핵심.

이에 TeaRAG를 도입한다 — 추론 스텝 간결성과 검색 콘텐츠 밀도를 동시 최적화하는 토큰 효율적 프레임워크. LLM이 핵심 개체를 식별·하위 질문으로 분해하고, 청크 의미 검색과 함께 간결한 지식 트리플 그래프 검색을 수행하며, 의미 유사성·공기로 지식 연관 그래프(KAG)를 구축하고 PPR로 중복·무관 청크를 걸러내며 고밀도 트리플로 보완한다. 추론 간결성을 위해 2단계 훈련(SFT로 추론 형식 습득 → IP-DPO로 과정 보상 기반 반복 최적화)을 제안한다. IP-DPO 과정 보상은 하위질의 생성·문맥 검색·요약 세 차원의 증거 획득을 지식 매칭으로 평가하고 스텝당 정보 이득으로 정규화한다.

## 한 줄 핵심
에이전트형 RAG의 토큰 낭비를, 그래프(트리플+PPR) 검색으로 검색 콘텐츠를 압축하고 과정 인식 DPO(IP-DPO)로 추론 스텝을 줄여 동시에 해결하는 토큰 효율 프레임워크.
