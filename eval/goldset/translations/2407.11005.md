## 원문 (abstract + intro)

Abstract
Retrieval-Augmented Generation (RAG) has become a standard architectural pat-
tern for incorporating domain-specific knowledge into user-facing chat applica-
tions powered by Large Language Models (LLMs). RAG systems are charac-
terized by (1) a document retriever that queries a domain-specific corpus for
context information relevant to an input query, and (2) an LLM that generates
a response based on the provided query and context.
However, comprehen-
sive evaluation of RAG systems remains a challenge due to the lack of unified
evaluation criteria and annotated datasets. In response, we introduce RAGBench:
the first comprehensive, large-scale RAG benchmark dataset of 100k examples.
It covers five unique industry-specific domains and various RAG task types.
RAGBench examples are sourced from industry corpora such as user manuals,
making it particularly relevant for industry applications. Further, we formalize the
TRACe evaluation framework: a set of explainable and actionable RAG evalua-
tion metrics applicable across all RAG domains. We release the labeled dataset
at https://huggingface.co/datasets/rungalileo/ragbench. RAGBench
explainable labels facilitate holistic evaluation of RAG systems, enabling action-
able feedback for continuous improvement of production applications. Thorough
extensive benchmarking, we find that LLM-based RAG evaluation methods strug-
gle to compete with a finetuned RoBERTa model on the RAG evaluation task. We
identify areas where existing approaches fall short and propose the adoption of
RAGBench with TRACe towards advancing the state of RAG evaluation systems.

Introduction
Despite remarkable reasoning and conversational abilities, out-of-the-box pre-trained Large Language
Models (LLMs) struggle to reason about out-of-domain, knowledge-intensive queries [20, 13]. In
response, Retriever-Augmented Generation (RAG) systems [20, 19] are becoming increasingly
popular in user-facing dialogue applications [34]. Generally, RAG systems comprise a retriever
component that queries relevant documents from an in-domain corpus and a downstream LLM
generator model that incorporates the retrieved documents along with the original user query to output
an informed response. The additional context helps ground the LLM in factual information and has
been shown to boost performance on knowledge-intensive tasks [20].
Still, when used in production settings, RAG systems are prone to hallucinations as the generator
model struggles to retrieve relevant information from the context [1, 30, 7]. In the absence of a
one-fits-all approach, application-specific RAG systems must be fine-tuned for optimal performance
on domain-specific tasks. However, the choice of retriever and generator models for each application
is complex and has serious implications on overall system quality and costs. With numerous

commercial and open-source generative LLMs readily available1 and many variable parameters in the
RAG system design (Figure 1), tuning an optimal system for a particular RAG application involves
iterative evaluation of multiple configurations. This motivates the need for automated RAG evaluation
solutions.
In response, automated RAG evaluation systems like RAGAS [9] and TruLens [36] have emerged.
These systems adopt a zero-shot LLM prompt-based approach to predict a set of curated RAG
evaluation metrics. However, the lack of unified RAG benchmarks makes it difficult to compare
approaches against each other. Each new study designs a new dataset, often employing LLMs as
generators and labelers [9, 32, 4], which renders them irreproducible. A few benchmarks like RGB
[4], AttributionBench [22] and RAGTruth [40] have been proposed recently, but they are small in
size and target a disjoint set of labels. The exact RAG evaluation criteria also vary from study to
study. ARES [32] and RAGAS [9] define a context relevance metric to evaluate the quality of the
retrieved documents, along with answer relevance and faithfulness to evaluate the quality of the
generative model. However, others have explored other metrics like correctness [1] noise rejection
and robustness [4], to name a few. Finally, most studies evaluate on small in-domain evaluation
datasets that are specific to each new application [32, 33, 9, 1, 4], leaving cross-domain generalization
an open question.
In this work we propose RAGBench: a comprehensive dataset for training and benchmarking
RAG evaluation models. RAGBench comprises data sourced from multiple domains along with
a comprehensive suite of evaluation metrics. Specifically, we adopt existing metric definitions for
context relevance, answer faithfulness [9, 32] and introduce two new metrics: context utilization and
answer completeness. We argue that this new suite of metrics better describes the overall RAG system
performance, with the potential to provide granular, actionable insights to the RAG practitioner.
We evaluate state-of-the art LLMs and existing RAG evaluation systems on RAGBench. We find that
a 400M-parameter DeBERTa-large model that was fine-tuned on RAGBench outperforms few-shot
LLM judges across numerous domains and task types. We highlight this result to motivate future
work aimed at leveraging these data for advancing RAG evaluation models and improving on the
proposed benchmark.


## 한글 번역

초록

검색증강생성(RAG)은 대규모 언어 모델(LLM)로 구동되는 사용자 대면 채팅 애플리케이션에 도메인 특화 지식을 통합하기 위한 표준 아키텍처 패턴이 되었다. RAG 시스템은 (1) 입력 질의와 관련된 맥락 정보를 위해 도메인 특화 코퍼스를 질의하는 문서 검색기(document retriever)와, (2) 제공된 질의와 맥락을 바탕으로 응답을 생성하는 LLM으로 특징지어진다. 그러나 통일된 평가 기준과 주석이 달린 데이터셋의 부재로 인해 RAG 시스템의 종합적 평가는 여전히 과제로 남아 있다. 이에 대응하여, 우리는 RAGBench를 소개한다: 이는 10만 개의 예제로 구성된 최초의 종합적이고 대규모인 RAG 벤치마크 데이터셋이다. 이 데이터셋은 다섯 개의 고유한 산업 특화 도메인과 다양한 RAG 작업 유형을 포괄한다. RAGBench 예제는 사용자 매뉴얼과 같은 산업 코퍼스에서 출처를 두어, 특히 산업 응용과의 관련성이 높다. 더 나아가, 우리는 TRACe 평가 프레임워크를 정형화한다: 이는 모든 RAG 도메인에 걸쳐 적용 가능한, 설명 가능하고 실행 가능한 RAG 평가 지표의 집합이다. 우리는 라벨이 달린 데이터셋을 https://huggingface.co/datasets/rungalileo/ragbench 에서 공개한다. RAGBench의 설명 가능한 라벨은 RAG 시스템의 전체론적(holistic) 평가를 촉진하여, 프로덕션 애플리케이션의 지속적 개선을 위한 실행 가능한 피드백을 가능하게 한다. 철저하고 광범위한 벤치마킹을 통해, 우리는 LLM 기반 RAG 평가 방법이 RAG 평가 작업에서 파인튜닝된 RoBERTa 모델과 경쟁하는 데 어려움을 겪는다는 것을 발견한다. 우리는 기존 접근법이 부족한 영역을 식별하고, RAG 평가 시스템의 최신 수준을 발전시키기 위해 TRACe와 함께 RAGBench를 채택할 것을 제안한다.

1. 서론

뛰어난 추론 및 대화 능력에도 불구하고, 즉시 사용 가능한 사전학습된 대규모 언어 모델(LLM)은 도메인 외부의 지식 집약적 질의에 대해 추론하는 데 어려움을 겪는다 [20, 13]. 이에 대응하여, 검색증강생성(RAG) 시스템 [20, 19]은 사용자 대면 대화 애플리케이션에서 점점 더 인기를 얻고 있다 [34]. 일반적으로 RAG 시스템은 도메인 내 코퍼스에서 관련 문서를 질의하는 검색기(retriever) 구성요소와, 검색된 문서를 원본 사용자 질의와 함께 통합하여 정보에 기반한 응답을 출력하는 하류(downstream) LLM 생성기 모델로 구성된다. 추가적인 맥락은 LLM을 사실 정보에 기반(ground)하도록 돕고, 지식 집약적 작업에서 성능을 향상시키는 것으로 나타났다 [20].

그럼에도 불구하고, 프로덕션 환경에서 사용될 때 RAG 시스템은 생성기 모델이 맥락에서 관련 정보를 검색하는 데 어려움을 겪기 때문에 환각(hallucination)에 취약하다 [1, 30, 7]. 만능 접근법이 부재한 상황에서, 애플리케이션 특화 RAG 시스템은 도메인 특화 작업에서 최적의 성능을 내도록 파인튜닝되어야 한다. 그러나 각 애플리케이션을 위한 검색기 및 생성기 모델의 선택은 복잡하며, 전체 시스템 품질과 비용에 심각한 영향을 미친다. 수많은 상업용 및 오픈소스 생성 LLM이 손쉽게 이용 가능하고1 RAG 시스템 설계에 많은 가변적 매개변수가 존재하는 상황에서(그림 1), 특정 RAG 애플리케이션을 위한 최적의 시스템을 튜닝하는 것은 다수의 구성에 대한 반복적 평가를 수반한다. 이는 자동화된 RAG 평가 솔루션의 필요성을 동기화한다.

이에 대응하여, RAGAS [9]와 TruLens [36]와 같은 자동화된 RAG 평가 시스템이 등장했다. 이 시스템들은 엄선된 RAG 평가 지표 집합을 예측하기 위해 제로샷(zero-shot) LLM 프롬프트 기반 접근법을 채택한다. 그러나 통일된 RAG 벤치마크의 부재는 접근법들을 서로 비교하기 어렵게 만든다. 각각의 새로운 연구는 새로운 데이터셋을 설계하며, 종종 LLM을 생성기 및 라벨러로 사용하는데 [9, 32, 4], 이는 그것들을 재현 불가능하게 만든다. RGB [4], AttributionBench [22], RAGTruth [40]와 같은 몇몇 벤치마크가 최근 제안되었지만, 그것들은 규모가 작고 서로 겹치지 않는 라벨 집합을 대상으로 한다. 정확한 RAG 평가 기준 또한 연구마다 다르다. ARES [32]와 RAGAS [9]는 검색된 문서의 품질을 평가하기 위한 맥락 관련성(context relevance) 지표와, 생성 모델의 품질을 평가하기 위한 답변 관련성(answer relevance) 및 충실성(faithfulness)을 정의한다. 그러나 다른 연구들은 정확성(correctness) [1], 노이즈 거부(noise rejection) 및 견고성(robustness) [4] 등 몇 가지 다른 지표를 탐구했다. 마지막으로, 대부분의 연구는 각각의 새로운 애플리케이션에 특화된 소규모 도메인 내 평가 데이터셋에서 평가하여 [32, 33, 9, 1, 4], 교차 도메인 일반화는 미해결 문제로 남겨 둔다.

본 연구에서 우리는 RAGBench를 제안한다: 이는 RAG 평가 모델의 학습 및 벤치마킹을 위한 종합적 데이터셋이다. RAGBench는 다수의 도메인에서 출처를 둔 데이터와 함께 종합적인 평가 지표 모음으로 구성된다. 구체적으로, 우리는 맥락 관련성과 답변 충실성에 대한 기존 지표 정의를 채택하고 [9, 32], 두 가지 새로운 지표를 도입한다: 맥락 활용도(context utilization)와 답변 완전성(answer completeness)이다. 우리는 이 새로운 지표 모음이 전체 RAG 시스템 성능을 더 잘 기술하며, RAG 실무자에게 세분화되고 실행 가능한 통찰을 제공할 잠재력이 있다고 주장한다.

우리는 최신 LLM과 기존 RAG 평가 시스템을 RAGBench에서 평가한다. 우리는 RAGBench로 파인튜닝된 4억 개 매개변수의 DeBERTa-large 모델이 다수의 도메인과 작업 유형에 걸쳐 퓨샷(few-shot) LLM 판정자(judge)를 능가함을 발견한다. 우리는 이 결과를 강조하여, 이 데이터를 활용해 RAG 평가 모델을 발전시키고 제안된 벤치마크를 개선하는 것을 목표로 하는 향후 연구를 동기화하고자 한다.
