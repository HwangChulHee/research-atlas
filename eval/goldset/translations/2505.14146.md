## 원문 (abstract + intro)

Abstract
Retrieval-augmented generation (RAG) sys-
tems empower large language models (LLMs)
to access external knowledge during infer-
ence. Recent advances have enabled LLMs
to act as search agents via reinforcement learn-
ing (RL), improving information acquisition
through multi-turn interactions with retrieval
engines. However, existing approaches either
optimize retrieval using search-only metrics
(e.g., NDCG) that ignore downstream utility or
fine-tune the entire LLM to jointly reason and
retrieve—entangling retrieval with generation
and limiting the real search utility and compat-
ibility with frozen or proprietary models. In
this work, we propose s3, a lightweight, model-
agnostic framework that decouples the searcher
from the generator and trains the searcher using
a Gain Beyond RAG reward: the improvement
in generation accuracy over naïve RAG. s3 re-
quires only 2.4k training samples to outperform
baselines trained on over 70× more data, con-
sistently delivering stronger downstream perfor-
mance across six general QA and five medical
QA benchmarks.1

the reward. While this approach improves answer

and generation makes it difficult to isolate genuine
retrieval improvements (see Figure 3). Moreover,
EM is a brittle reward signal—failing to reward
semantically correct answers phrased differently.
This motivates a shift toward a modular frame-
work where search and generation are cleanly sep-
arated, and optimization focuses purely on search
quality with respect to downstream utility (Dai
et al., 2025). We propose s3, a simple yet powerful
framework that trains a search-only agent using
a novel reward signal: Gain Beyond RAG (GBR).
GBR measures how much better the generator per-
forms when conditioned on retrieved documents
from s3, compared to naive top-k retrieval. This
setup keeps the generator LLM frozen, sidesteps
answer token overfitting, and directly optimizes the
retrieval component to serve any black-box LLM.
Remarkably, s3 achieves strong gains with only
2.4k training examples, outperforming DeepRe-
trieval (focused on retrieval metrics) and Search-R1
(entangled optimization) both in terms of context
quality and final answer performance.
Our main contributions are:
• We introduce s3, a modular, RL-based search
framework that optimizes for generation quality
without touching the generator.
• We define Gain Beyond RAG (GBR), a princi-
pled, model-agnostic reward signal that quanti-
fies improvements over standard retrieval.
• We show that s3 outperforms state-of-the-art
agentic RAG methods on six general and five
medical QA benchmarks, using 70× less train-
ing data (see Figure 1).


## 한글 번역

초록

검색증강생성(RAG) 시스템은 대규모 언어모델(LLM)이 추론 시점에 외부 지식에 접근할 수 있게 해준다. 최근의 발전은 강화학습(RL)을 통해 LLM이 검색 에이전트로 동작할 수 있게 하여, 검색 엔진과의 다중 턴 상호작용을 통한 정보 획득을 개선했다. 그러나 기존 접근법들은 하류(downstream) 효용을 무시한 채 검색 전용 지표(예: NDCG)로 검색을 최적화하거나, LLM 전체를 미세조정(fine-tune)하여 추론과 검색을 함께 수행하게 한다. 후자는 검색을 생성과 얽히게 만들어 실제 검색 효용을 제한하고, 동결된(frozen) 모델이나 독점(proprietary) 모델과의 호환성을 떨어뜨린다. 본 연구에서 우리는 검색기(searcher)를 생성기(generator)로부터 분리하고, Gain Beyond RAG 보상—단순 RAG 대비 생성 정확도의 향상분—을 사용하여 검색기를 학습시키는 경량의 모델 비종속(model-agnostic) 프레임워크인 s3를 제안한다. s3는 단 2.4k개의 학습 표본만으로도 70배 이상 많은 데이터로 학습된 베이스라인들을 능가하며, 6개의 일반 QA 벤치마크와 5개의 의료 QA 벤치마크 전반에서 일관되게 더 강력한 하류 성능을 제공한다.1

1. 서론

검색증강생성(RAG)은 대규모 언어모델(LLM)이 관련 문서를 검색하고 그에 조건화하여 생성함으로써 외부 지식에 접근하고 추론할 수 있게 해준다 (Lewis et al., 2020). 그림 2에 나타난 것처럼, 우리는 RAG 시스템의 진화를 세 단계로 분류한다.

고전적 RAG(Classic RAG). 초기 접근법들은 정적 검색 방법에 의존했으며, 여기서 질의는 고정되어 있었고 검색 품질은 하류 생성 성능과 분리되어 있었다. 이러한 시스템들은 단순함에도 불구하고, 맥락적 또는 다중 홉(multi-hop) 추론이 필요한 질의에서는 흔히 저조한 성능을 보였다.

그림 1: 6개의 일반 QA 데이터셋과 5개의 의료 QA 데이터셋 전반에 걸친 학습 데이터량 대 평균 성능 (생성 LLM으로 Claude-3-Haiku를 사용하여 테스트함).

RLVR 이전(Pre-RLVR). 검색 품질을 개선하기 위해, 후속 방법들은 추론 시점에 LLM이 더 능동적으로 참여하도록 했다. 능동적 RAG(Active RAG) 기법들 (Yao et al., 2022; Jiang et al., 2023; Trivedi et al., 2023a)은 질의 생성, 검색, 추론을 다중 턴 루프 안에서 교차로 수행했다. 이러한 시스템들은 반복적 검색을 도입했지만 일반적으로 제로샷(zero-shot) 프롬프팅에 의존했고 학습 가능한 구성요소를 갖추지 못했다. Self-RAG (Asai et al., 2023)는 이러한 행동들을 더 큰 모델로부터 더 작은 모델로 지도 미세조정(supervised fine-tuning)을 통해 증류(distill)하여, 작은 모델이 외부 보상 없이도 효과적으로 추론하고 검색하도록 가르쳤다. 이러한 방법들은 유연성을 개선하고 감독 비용을 줄였지만, 여전히 결과 신호를 사용하여 검색을 최적화하지는 못했다.

RLVR 시기(RLVR Period). 최근 검증 가능한 보상을 활용한 강화학습(RLVR) (Su et al., 2025)의 등장은 새로운 단계를 의미한다. DeepSeek-R1-Zero (Guo et al., 2025)는 규칙 기반의 결과 주도형 보상(예: 정답 정확도)만으로도 강력한 추론 에이전트를 학습시킬 수 있음을 보였다. 이 아이디어를 기반으로, DeepRetrieval (Jiang et al., 2025)은 재현율(recall)이나 NDCG 같은 검색 지향 지표를 사용하여 질의 생성기를 학습시키는 데 RL을 적용했다. 그러나 이러한 지표들은 하류 정답 품질과 단절되어 있다. Search-R1 (Jin et al., 2025)은 정확 일치(exact match, EM)를 보상으로 사용하여 단일 모델이 강화학습을 통해 검색과 생성을 함께 수행하도록 학습시켰다. 이 접근법은 정답 정확도를 개선하지만, 검색과 생성이 긴밀하게 얽혀 있어 진정한 검색 개선분을 분리해내기 어렵다(그림 3 참조). 게다가 EM은 깨지기 쉬운(brittle) 보상 신호로, 다르게 표현된 의미상 올바른 정답에 보상을 주지 못한다.

그림 2: RAG는 고정형 또는 지도형 검색에서 RL 기반 에이전트형 방법으로 발전해 왔다. 선행 연구가 검색이나 생성을 함께 학습시키는 반면, s3는 오직 검색기에만 집중하여 생성기 LLM을 조정하지 않고도 생성을 개선한다.

그림 3: 에이전트형 RAG의 분해. 종단간(end-to-end) 접근법은 전체 생성 정확도를 사용하여 모델 전체를 미세조정하므로 검색의 기여를 분리하기 어렵다. 이와 대조적으로 s3는 생성기를 동결하고, 검색된 맥락이 단순 RAG 대비 더해주는 가치를 정량화하는 새로운 보상인 Gain Beyond RAG(GBR)로 검색기만을 학습시켜, 모듈식이고 효율적인 최적화를 가능하게 한다.

이는 검색과 생성이 깔끔하게 분리되고, 최적화가 하류 효용에 대한 검색 품질에만 순수하게 집중하는 모듈식 프레임워크로의 전환을 동기화한다 (Dai et al., 2025). 우리는 새로운 보상 신호인 Gain Beyond RAG(GBR)를 사용하여 검색 전용(search-only) 에이전트를 학습시키는 단순하지만 강력한 프레임워크 s3를 제안한다. GBR은 단순 top-k 검색과 비교했을 때, s3가 검색한 문서에 조건화되었을 때 생성기가 얼마나 더 잘 수행하는지를 측정한다. 이 구성은 생성기 LLM을 동결된 상태로 유지하고, 정답 토큰 과적합(overfitting)을 회피하며, 검색 구성요소를 직접 최적화하여 어떤 블랙박스 LLM에도 서비스하도록 한다. 놀랍게도 s3는 단 2.4k개의 학습 예시만으로 강력한 성능 향상을 달성하여, (검색 지표에 집중하는) DeepRetrieval과 (얽힌 최적화를 수행하는) Search-R1을 맥락 품질과 최종 정답 성능 양쪽에서 모두 능가한다.

우리의 주요 기여는 다음과 같다:
• 우리는 생성기를 건드리지 않고도 생성 품질을 최적화하는 모듈식 RL 기반 검색 프레임워크 s3를 소개한다.
• 우리는 표준 검색 대비 개선분을 정량화하는 원리적이고 모델 비종속적인 보상 신호인 Gain Beyond RAG(GBR)를 정의한다.
• 우리는 s3가 70배 적은 학습 데이터를 사용하면서도 6개의 일반 QA 벤치마크와 5개의 의료 QA 벤치마크에서 최첨단(state-of-the-art) 에이전트형 RAG 방법들을 능가함을 보인다(그림 1 참조).
