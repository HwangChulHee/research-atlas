## 원문 (abstract + intro)

Abstract
Efficiently acquiring external knowledge and up-to-date information is
essential for effective reasoning and text generation in large language
models (LLMs).
Prompting advanced LLMs with reasoning capabili-
ties to use search engines during inference is often suboptimal, as the
LLM might not fully possess the capability on how to interact optimally
with the search engine.
This paper introduces SEARCH-R1, an exten-
sion of reinforcement learning (RL) for reasoning frameworks where
the LLM learns to autonomously generate (multiple) search queries dur-
ing step-by-step reasoning with real-time retrieval.
SEARCH-R1 opti-
mizes LLM reasoning trajectories with multi-turn search interactions,
leveraging retrieved token masking for stable RL training and a simple
outcome-based reward function. Experiments on seven question-answering
datasets show that SEARCH-R1 improves performance by 24% (Qwen2.5-
7B) and 20% (Qwen2.5-3B) over various RAG baselines under the same
setting. This paper further provides empirical insights into RL optimiza-
tion methods, LLM choices, and response length dynamics in retrieval-
augmented reasoning. The code and model checkpoints are available at
https://github.com/PeterGriffinJin/Search-R1.

Introduction
Large language models (LLMs) have demonstrated remarkable capabilities in natural
language understanding and generation (Hendrycks et al., 2020; Clark et al., 2018). Despite
these achievements, LLMs often encounter challenges when tasked with complex reasoning
(Wei et al., 2022) and retrieving up-to-date information from external sources (Jin et al.,
2024). Addressing these limitations necessitates integrating advanced reasoning abilities
(Huang & Chang, 2022) and the capability to interact effectively with search engines to best
utilize external up-to-date information (Schick et al., 2023).
Existing approaches for integrating LLMs with search engines typically fall into two cat-
egories: (1) retrieval-augmented generation (RAG) (Gao et al., 2023; Lewis et al., 2020)
and (2) treating the search engine as a tool (Yao et al., 2023; Schick et al., 2023). RAG
models often retrieve passages based on the LLM input as query and incorporate them
into the LLM’s context for generation (Lewis et al., 2020). This allows the LLM to leverage
external knowledge when answering questions. Although existing work (Trivedi et al.,
2022a) prompts LLM for multi-turn, multi-query retrieval, this approach is suboptimal
because the LLM is not optimized to learn how to interact effectively with search engines
during training. Alternatively, LLMs can be prompted or trained to utilize tools, including
search engines, as part of their reasoning process (Qu et al., 2025; Trivedi et al., 2022a).
However, prompting-based approaches often struggle to generalize, as certain tasks may
not have been encountered during LLM pretraining. On the other hand, training-based

approaches offer greater adaptability but are difficult to scale effectively due to their reliance
on large-scale, high-quality annotated trajectories and the inherent non-differentiability
of the search operation, which renders end-to-end gradient descent-based optimization
inapplicable (Schick et al., 2023; Asai et al., 2024).
Reinforcement Learning (RL) (Sutton et al., 1999; Kaelbling et al., 1996) has emerged as a
potent paradigm for enhancing the reasoning capabilities of LLMs (Guo et al., 2025; Hou
et al., 2025; Xie et al., 2025; Kumar et al., 2024). Notably, models like OpenAI-o1 (Jaech et al.,
2024) and DeepSeek-R1 (Guo et al., 2025) have leveraged RL techniques (e.g., PPO (Schulman
et al., 2017) and GRPO (Shao et al., 2024)) to improve logical inference and problem-solving
skills by learning from experience and feedback. With RL, even when trained solely on
the outcome rewards, the models learn complex reasoning capabilities, including self-
verification (Weng et al., 2022) and self-correction (Kumar et al., 2024). However, applying
RL to search-and-reasoning scenarios presents three key challenges: (1) RL Framework and
Stability – It remains unclear how to effectively integrate the search engine into the RL
approaches for LLMs while ensuring stable optimization, particularly when incorporating
retrieved context. (2) Multi-Turn Interleaved Reasoning and Search – Ideally, the LLM
should be capable of iterative reasoning and search engine calls, dynamically adjusting the
retrieval strategy based on the complexity of the problem. (3) Reward Design – Designing
an effective reward function for search and reasoning tasks remains a fundamental challenge,
as it is unclear whether simple outcome-based rewards are sufficient to guide the LLM to
learn meaningful and consistent search behaviors.
To address aforementioned challenges, we introduce SEARCH-R1, a novel RL framework
that enables LLMs to interact with search engines in an interleaved manner with their own
reasoning. Specifically, SEARCH-R1 introduces the following key innovations: (1) We model
the search engine as part of the environment, enabling sampled trajectory sequences that
interleave LLM token generation with search engine retrievals. SEARCH-R1 is compatible
with various RL algorithms, including PPO and GRPO, and we apply retrieved token
masking to ensure stable optimization. (2) SEARCH-R1 supports multi-turn retrieval and
reasoning, invoking search calls when explicitly triggered by <search> and </search>
tokens. Retrieved content is enclosed within <information> and </information> tokens,
while LLM reasoning steps are wrapped within <think> and </think> tokens. The final
answer is formatted using <answer> and </answer> tokens, allowing for structured, iterative
decision-making. (3) We adopt a straightforward outcome-based reward function, avoiding
the complexity of process-based rewards. Our results demonstrate that this minimal reward
design is effective in search-and-reasoning scenarios. As such, SEARCH-R1 can be viewed as
an extension of DeepSeek-R1 Zero (Guo et al., 2025), which primarily focuses on parametric
reasoning by introducing search-augmented RL training for enhanced retrieval-driven
decision-making.
In summary, our key contributions are threefold:
• Our work analyzes the challenges and provides perspectives on implementing RL to
improve how LLMs reason using search engine results.
• We propose SEARCH-R1, a novel RL framework that supports LLM rollouts and direct
optimization with a search engine, including retrieved token masking to stabilize RL
training, multi-turn interleaved reasoning and search to support complex task-solving
and an effective outcome reward function.
• We conduct systematic experiments to demonstrate the effectiveness of SEARCH-R1, with
two LLMs achieving respective average relative improvements of 41% and 20% over
RAG baselines under the same experimental setup (e.g., same retrieval model, training
data, and pre-trained LLMs). In addition, we provide insights on RL for reasoning and
search settings, including RL method selection, different LLM choices, and response
length study.


## 한글 번역

초록

외부 지식과 최신 정보를 효율적으로 획득하는 것은 거대 언어모델(LLM)의 효과적인 추론과 텍스트 생성에 필수적이다. 추론 능력을 갖춘 고급 LLM에게 추론 시점(inference)에 검색엔진을 사용하도록 프롬프팅하는 방식은 흔히 최적이 아니다. LLM이 검색엔진과 최적으로 상호작용하는 방법을 충분히 갖추지 못했을 수 있기 때문이다. 본 논문은 SEARCH-R1을 소개한다. 이는 추론 프레임워크를 위한 강화학습(RL)의 확장으로, LLM이 실시간 검색(real-time retrieval)을 동반한 단계별 추론 도중 (여러 개의) 검색 질의를 자율적으로 생성하도록 학습한다. SEARCH-R1은 다중턴 검색 상호작용으로 LLM의 추론 궤적을 최적화하며, 안정적인 RL 훈련을 위해 검색된 토큰 마스킹(retrieved token masking)과 단순한 결과 기반(outcome-based) 보상 함수를 활용한다. 일곱 개 질의응답 데이터셋 실험에서 SEARCH-R1은 동일 설정의 여러 RAG 베이스라인 대비 성능을 24%(Qwen2.5-7B)와 20%(Qwen2.5-3B) 향상시켰다. 본 논문은 또한 검색증강 추론(retrieval-augmented reasoning)에서의 RL 최적화 방법, LLM 선택, 응답 길이 동역학에 대한 실증적 통찰을 제공한다. 코드와 모델 체크포인트는 https://github.com/PeterGriffinJin/Search-R1 에서 제공한다.

1. 서론

거대 언어모델(LLM)은 자연어 이해와 생성에서 놀라운 능력을 보여왔다(Hendrycks et al., 2020; Clark et al., 2018). 이러한 성취에도 불구하고 LLM은 복잡한 추론(Wei et al., 2022)과 외부 출처로부터 최신 정보를 검색하는 일(Jin et al., 2024)에서 종종 어려움에 부딪힌다. 이러한 한계를 해결하려면 고급 추론 능력(Huang & Chang, 2022)과, 외부의 최신 정보를 가장 잘 활용하기 위해 검색엔진과 효과적으로 상호작용하는 능력(Schick et al., 2023)을 통합해야 한다.

LLM을 검색엔진과 통합하는 기존 접근은 보통 두 범주로 나뉜다. (1) 검색증강생성(RAG)(Gao et al., 2023; Lewis et al., 2020)과 (2) 검색엔진을 도구(tool)로 취급하는 방식(Yao et al., 2023; Schick et al., 2023)이다. RAG 모델은 흔히 LLM 입력을 질의로 삼아 구절을 검색하고 이를 LLM의 문맥에 포함시켜 생성한다(Lewis et al., 2020). 이로써 LLM은 질문에 답할 때 외부 지식을 활용할 수 있다. 기존 연구(Trivedi et al., 2022a)가 다중턴·다중질의 검색을 위해 LLM을 프롬프팅하긴 하지만, 이 접근은 훈련 도중 LLM이 검색엔진과 효과적으로 상호작용하는 법을 학습하도록 최적화되지 않았기에 최적이 아니다. 대안으로, LLM은 추론 과정의 일부로 검색엔진을 포함한 도구를 활용하도록 프롬프팅되거나 훈련될 수 있다(Qu et al., 2025; Trivedi et al., 2022a). 그러나 프롬프팅 기반 접근은 일반화에 자주 어려움을 겪는데, 특정 과제가 LLM 사전학습 중에 마주친 적이 없을 수 있기 때문이다. 반면 훈련 기반 접근은 더 큰 적응성을 제공하지만, 대규모의 고품질 주석 궤적에 의존하고 검색 연산이 본질적으로 미분 불가능(non-differentiable)하여 종단간(end-to-end) 경사하강 기반 최적화가 적용 불가능하기에 효과적으로 확장하기 어렵다(Schick et al., 2023; Asai et al., 2024).

강화학습(RL)(Sutton et al., 1999; Kaelbling et al., 1996)은 LLM의 추론 능력을 강화하는 강력한 패러다임으로 떠올랐다(Guo et al., 2025; Hou et al., 2025; Xie et al., 2025; Kumar et al., 2024). 특히 OpenAI-o1(Jaech et al., 2024)과 DeepSeek-R1(Guo et al., 2025) 같은 모델들은 RL 기법(예: PPO(Schulman et al., 2017)와 GRPO(Shao et al., 2024))을 활용해 경험과 피드백으로부터 학습함으로써 논리적 추론과 문제해결 능력을 향상시켰다. RL을 사용하면, 오직 결과 보상만으로 훈련하더라도 모델은 자기검증(self-verification)(Weng et al., 2022)과 자기교정(self-correction)(Kumar et al., 2024)을 포함한 복잡한 추론 능력을 학습한다. 그러나 RL을 검색-추론 시나리오에 적용하는 데는 세 가지 핵심 과제가 있다. (1) RL 프레임워크와 안정성 – 특히 검색된 문맥을 포함할 때, 안정적 최적화를 보장하면서 검색엔진을 LLM용 RL 접근에 효과적으로 통합하는 방법이 아직 불분명하다. (2) 다중턴 교차(interleaved) 추론과 검색 – 이상적으로 LLM은 반복적인 추론과 검색엔진 호출이 가능해야 하며, 문제의 복잡도에 따라 검색 전략을 동적으로 조정해야 한다. (3) 보상 설계 – 단순한 결과 기반 보상이 LLM으로 하여금 의미 있고 일관된 검색 행동을 학습하도록 안내하기에 충분한지가 불분명하므로, 검색·추론 과제를 위한 효과적인 보상 함수를 설계하는 일은 여전히 근본적인 과제다.

위 과제들을 해결하기 위해 우리는 LLM이 자신의 추론과 교차된 방식으로 검색엔진과 상호작용할 수 있게 하는 새로운 RL 프레임워크 SEARCH-R1을 도입한다. 구체적으로 SEARCH-R1은 다음의 핵심 혁신을 제시한다. (1) 우리는 검색엔진을 환경(environment)의 일부로 모델링하여, LLM 토큰 생성과 검색엔진 검색을 교차하는 샘플링된 궤적 시퀀스를 가능하게 한다. SEARCH-R1은 PPO, GRPO를 포함한 다양한 RL 알고리즘과 호환되며, 안정적 최적화를 위해 검색된 토큰 마스킹을 적용한다. (2) SEARCH-R1은 다중턴 검색과 추론을 지원하며, <search>와 </search> 토큰으로 명시적으로 촉발될 때 검색 호출을 부른다. 검색된 내용은 <information>과 </information> 토큰으로 감싸고, LLM 추론 단계는 <think>와 </think> 토큰으로 감싸며, 최종 답은 <answer>와 </answer> 토큰으로 형식화하여 구조적·반복적 의사결정을 가능하게 한다. (3) 우리는 과정 기반(process-based) 보상의 복잡성을 피하고 직관적인 결과 기반 보상 함수를 채택한다. 우리 결과는 이 최소한의 보상 설계가 검색-추론 시나리오에서 효과적임을 입증한다. 이런 점에서 SEARCH-R1은, 주로 파라메트릭 추론에 초점을 둔 DeepSeek-R1 Zero(Guo et al., 2025)에, 검색 기반 의사결정을 강화하기 위한 검색증강 RL 훈련을 도입해 확장한 것으로 볼 수 있다.

요약하면 우리의 핵심 기여는 세 가지다.
• 우리 연구는 LLM이 검색엔진 결과를 사용해 추론하는 방식을 개선하기 위한 RL 구현의 과제를 분석하고 관점을 제시한다.
• 우리는 SEARCH-R1을 제안한다. 이는 검색엔진을 통한 LLM 롤아웃과 직접 최적화를 지원하는 새로운 RL 프레임워크로, RL 훈련을 안정화하는 검색된 토큰 마스킹, 복잡한 과제 해결을 지원하는 다중턴 교차 추론·검색, 효과적인 결과 보상 함수를 포함한다.
• 우리는 SEARCH-R1의 효과를 입증하는 체계적 실험을 수행하며, 두 LLM이 동일한 실험 설정(예: 동일 검색 모델, 훈련 데이터, 사전학습 LLM)에서 RAG 베이스라인 대비 각각 평균 상대 향상 41%와 20%를 달성한다. 또한 RL 방법 선택, 다양한 LLM 선택, 응답 길이 연구를 포함해 추론·검색 설정에서의 RL에 대한 통찰을 제공한다.
