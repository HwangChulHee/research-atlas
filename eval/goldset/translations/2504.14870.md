## 원문 (abstract + intro)

Abstract
Tool-integrated reasoning (TIR) augments large language models (LLMs) with
the ability to invoke external tools during long-form reasoning, such as search
engines and code interpreters, to solve tasks beyond the capabilities of internal
reasoning. While reinforcement learning (RL) has shown promise in training such
agents, most of existing approaches typically optimize only for final correctness
without considering the efficiency or necessity of external tool use. This often leads
to excessive tool calling, incurring high computational costs and hindering the
development of internal reasoning capabilities - a phenomenon known as cognitive
offloading. To this end, we propose Optimal Tool Call-controlled Policy Optimiza-
tion (OTC-PO), a simple yet effective RL-based framework that encourages models
to produce accurate answers with minimal tool calls. Our method introduces a
tool-integrated reward that jointly considers answer correctness and corresponding
tool use behavior of model to reach that answer. To validate the effectiveness, we
introduce the metric of tool productivity, defined as the ratio between the number
of correct answers and the total number of tool calls across all test cases. This
metric reflects how efficiently tool usage contributes to successful task completion,
with higher values indicating smarter and more autonomous reasoning. We instan-
tiate this framework within both Proximal Policy Optimization (PPO) and Group
Relative Preference Optimization (GRPO), resulting in OTC-PPO and OTC-GRPO.
Experiments with Qwen-2.5 and Qwen-Math across multiple QA benchmarks show
that our approach reduces tool calls by up to 68.3% and improves tool productivity
by up to 215.4%, while maintaining comparable answer accuracy.

While recent efforts have sought to extend RL to TIR by leveraging rule-based rewards tied to final
answer correctness [14–16], these approaches fall short in modeling the optimal tool use behavior of
model to arrive at such answer. Despite achieving strong performance and improved generalization,
current methods often encourage indiscriminate tool usage, introducing critical challenges during
both training and inference. On the one hand, frequent and unnecessary tool calls incur substantial
computational and temporal overhead, as executing tools - especially at scale - demands significant
infrastructure and increases operational costs, limiting deployment feasibility. On the other hand,
extensively rely on external tools may hinder the development and utilization of its own internal
reasoning capabilities — a phenomenon commonly known as cognitive offloading [17]. Fig. 1
presents a representative example. The baseline method (i.e., Search-R1 [14]), which optimizes
only for final answer accuracy, excessively invokes external tools, even when the model is capable
of solving the problem independently without any tools. By offloading reasoning to external tools,
the model bypasses opportunities to strengthen its internal reasoning abilities, thereby ultimately
undermining long-term generalization and autonomy 1.
Therefore, we argue that a truly autonomous agentic LLM should learn to produce correct answers or
complete tasks successfully with the minimal number of external tool calls - an objective that aligns
closely with Yann LeCun’s vision for autonomous machine intelligence [1]. We aim to develop such
model based on a foundational assumption: for each question and each model, there exists an optimal
number of tool calls, defined as the minimal number required for the model to arrive at a correct
answer. Crucially, this optimal number could be varied across models or questions. For the same
question, models with different reasoning or representation capabilities may require different levels
of tool assistance. Similarly, even for a single model, the number of necessary tool calls can vary
significantly depending on the complexity of different question. For example, some questions may be
solvable through internal reasoning alone - requiring zero tool calls2 - while others may necessitate
one or more tool interactions to yield a correct answer. This variability also highlights the unique
advantages of RL in steering the behavior of an agent on a per-question, per-model basis, rather than
a just one-fits-all method (i.e., SFT). Ideally, among all tool-integrated reasoning trajectories that
lead to the correct answer for the same question, the reward function in RL should assign the highest

We then propose Optimal Tool Calls controlled Policy Optimization (OTC-PO), a simple yet ef-
fective RL-based method that enables models to learn optimal tool use behavior besides providing
correct answer. OTC-PO trains models to minimize the number of tool calls required to reach a
correct solution, thereby reducing both training and inference costs without sacrificing accuracy. To
achieve this, we introduce a tool-integrated reward that modulates traditional reward signals, such
as correctness, with a scaling coefficient reflecting tool efficiency. This encourages the model to
prioritize correct answers that require fewer tool invocations. Our formulation shifts the optimization
objective from correctness alone to tool productivity, defined as the ratio between task benefit (e.g.,
answer accuracy) and tool usage cost (e.g., number of tool calls). To the best of our knowledge,
this is the pioneering RL-based framework that explicitly optimizes for both the efficiency and
effectiveness of tool-integrated reasoning. OTC-PO is lightweight and plug-and-play, requiring only
minimal changes to standard RL pipelines, making it easy to adopt in existing systems. We conduct
comprehensive experiments on two widely used tools: search and code based on several LLMs,
Qwen-2.5-3B/7B-Base and Qwen2.5-Math-1.5B/7B-Base [18], and derive models with a substantial
tool calls reduction in trajectory while approximately maintaining accuracy with strong baselines. In
summary, the key contributions are as follows.
• We are the pioneering study to i) consider tool use behavior beyond final answer correctness by
proposing a new task formulation that has been largely overlooked in prior work; ii) identify the
cognitive offloading phenomenon in TIR of LLMs; iii) introduce the concept of tool productivity to
measure the effectiveness and efficiency of TIR.
• We propose a simple, faster, scalable, and generalizable OTC-PO algorithm to encourage the
model to use optimal tool calls to solve the problem and maximize tool productivity, based on
the fundamental observation that there exists a minimal number of tool calls required for each
question and model pair. We note that it is compatible with various RL algorithms and can be easily
implemented.
• We implement OTC-PPO and OTC-GRPO as two typical methods without losing the adaptability
and generalization based on PPO [19] and GRPO [20] algorithms, and the experimental results
on several benchmarks and baselines demonstrate significant reductions in tool call cost while
preserving most of the accuracy in both in-domain and out-of-domain evaluation.


## 한글 번역

초록

도구통합추론(tool-integrated reasoning, TIR)은 거대언어모델(large language models, LLMs)에 장문 추론 과정에서 검색 엔진이나 코드 인터프리터와 같은 외부 도구를 호출하는 능력을 부여하여, 내부 추론만으로는 해결할 수 없는 과제를 풀 수 있게 한다. 강화학습(reinforcement learning, RL)이 이러한 에이전트를 학습시키는 데 가능성을 보여 왔지만, 기존 접근법 대부분은 외부 도구 사용의 효율성이나 필요성을 고려하지 않고 오직 최종 정답만을 최적화한다. 이는 흔히 과도한 도구 호출로 이어져 높은 계산 비용을 유발하고 내부 추론 능력의 발달을 저해하는데, 이러한 현상은 인지적 부하 이전(cognitive offloading)으로 알려져 있다. 이를 위해 우리는 최적 도구 호출 제어 정책 최적화(Optimal Tool Call-controlled Policy Optimization, OTC-PO)를 제안한다. 이는 모델이 최소한의 도구 호출로 정확한 답을 산출하도록 유도하는, 단순하지만 효과적인 RL 기반 프레임워크이다. 우리 방법은 정답의 정확성과 그 답에 도달하기 위한 모델의 도구 사용 행동을 함께 고려하는 도구통합보상(tool-integrated reward)을 도입한다. 효과를 검증하기 위해 우리는 도구 생산성(tool productivity)이라는 지표를 도입하는데, 이는 모든 테스트 사례에 걸친 정답 수와 전체 도구 호출 수의 비율로 정의된다. 이 지표는 도구 사용이 과제의 성공적 완료에 얼마나 효율적으로 기여하는지를 반영하며, 값이 높을수록 더 영리하고 자율적인 추론을 의미한다. 우리는 이 프레임워크를 근접 정책 최적화(Proximal Policy Optimization, PPO)와 그룹 상대 선호 최적화(Group Relative Preference Optimization, GRPO) 양쪽에 적용하여 OTC-PPO와 OTC-GRPO를 도출한다. 여러 QA 벤치마크에 걸쳐 Qwen-2.5와 Qwen-Math로 수행한 실험은 우리 접근법이 도구 호출을 최대 68.3%까지 줄이고 도구 생산성을 최대 215.4%까지 향상시키면서도 정답 정확도는 비슷한 수준으로 유지함을 보여준다.

1. 서론

"자율적 기계 지능은 어떤 과제를 학습하기 위해 시스템이 실세계에서 취해야 하는 행동의 수를 최소화하도록 설계된다." — Yann LeCun [1]

거대언어모델(LLMs)의 최근 발전은 단순한 규칙 기반 보상을 사용한 강화학습(RL)으로 미세조정될 때 인상적인 추론 능력을 보여주었다. 이는 OpenAI의 o1 [2], DeepSeek-R1 [3], QwQ [4]와 같은 거대추론모델(large reasoning models, LRMs)의 등장으로 이어졌으며, 이들은 확장된 연쇄적 사고(Chain-of-Thought, CoT) 추론 [5]에서 창의적이고 다양한 행동을 나타낸다. 이러한 모델들은 성찰과 분해와 같이 인간 인지를 연상시키는 메타 추론 능력을 발달시키는데, 이는 복잡한 과제에서 성능을 향상시키는 것으로 나타났다 [6–8]. 그러나 LRMs의 내부 추론 능력만으로는 최신 도메인 지식에 접근하거나 정밀한 계산을 수행하는 등 외부 환경과의 상호작용을 요구하는 과제에서 종종 불충분하거나 심지어 실현 불가능하다. 이러한 한계를 극복하기 위해 도구통합추론(tool-integrated reasoning, TIR)이 유망한 패러다임으로 부상했으며 [9–13], 이는 LLMs가 외부 도구와 연결될 수 있게 한다.

예를 들어 "주세페 베르디(Giuseppe Verdi)와 앙브루아즈 토마(Ambroise Thomas)가 둘 다 오페라 작곡가인가?"라는 질문(정답: 예)을 살펴보자. Qwen-7B-Base 모델에서 검색을 도구로 사용한 사례 연구에서, 네 가지 방법은 모두 정답에 도달하지만 내부 추론과 외부 도구 사용에서 서로 다른 행동을 보인다. OTC-PPO는 먼저 두 인물이 모두 오페라 작곡가인지 판단해야 한다고 생각한 뒤 한 번의 검색을 수행하고 그 결과에 근거해 "예"라고 답한다(도구 호출 = 1). 반면 베이스라인인 Search-R1-PPO는 두 인물을 각각 검색하고 다시 질문 전체를 검색하여 총 세 번의 도구 호출을 사용한다(도구 호출 = 3). Search-R1-GRPO는 두 인물과 그들의 작품을 한 번 검색하고 베르디를 다시 검색하여 두 번의 도구 호출을 사용한다(도구 호출 = 2). 이에 비해 OTC-GRPO는 외부 도구를 전혀 사용하지 않고, 베르디가 "리골레토", "라 트라비아타", "아이다" 등으로 유명한 이탈리아 오페라 작곡가이고 토마가 "미뇽", "햄릿" 등으로 알려진 프랑스 오페라 작곡가임을 내부 추론만으로 정리하여 "예"라고 답한다(도구 호출 = 0). 인지적 부하 이전(cognitive offloading)은 외부 도구에 대한 의존을 조장하고 기억 능력을 약화시키며 내부 추론의 발달을 저해한다.

검색 엔진, 코드 인터프리터, 심지어 물리적 로봇과 같은 외부 도구와 연결됨으로써, TIR은 언어 이해를 외부 능력과 연결하여 내부 지식만에 의존하는 추론을 넘어 LLMs의 문제 해결 범위를 크게 확장한다.

최근의 노력들은 최종 정답의 정확성에 연동된 규칙 기반 보상을 활용하여 RL을 TIR로 확장하고자 했으나 [14–16], 이러한 접근법들은 그러한 답에 도달하기 위한 모델의 최적 도구 사용 행동을 모델링하는 데에는 미흡하다. 강력한 성능과 향상된 일반화를 달성함에도 불구하고, 현재의 방법들은 종종 무분별한 도구 사용을 조장하여 학습과 추론 양쪽에서 중대한 난제를 야기한다. 한편으로, 빈번하고 불필요한 도구 호출은 상당한 계산적·시간적 부담을 초래하는데, 도구를 실행하는 일은 특히 대규모에서 상당한 인프라를 요구하고 운영 비용을 증가시켜 배포 실현 가능성을 제한한다. 다른 한편으로, 외부 도구에 광범위하게 의존하는 것은 모델 자신의 내부 추론 능력의 발달과 활용을 저해할 수 있는데, 이는 흔히 인지적 부하 이전(cognitive offloading)으로 알려진 현상이다 [17]. Fig. 1은 대표적인 예를 보여준다. 최종 정답 정확도만을 최적화하는 베이스라인 방법(즉, Search-R1 [14])은 모델이 어떤 도구 없이도 독립적으로 문제를 해결할 수 있는 경우에도 외부 도구를 과도하게 호출한다. 추론을 외부 도구로 떠넘김으로써 모델은 내부 추론 능력을 강화할 기회를 건너뛰게 되고, 따라서 궁극적으로 장기적 일반화와 자율성을 약화시킨다.

따라서 우리는 진정으로 자율적인 에이전트형 LLM이라면 최소한의 외부 도구 호출로 정답을 산출하거나 과제를 성공적으로 완료하는 법을 학습해야 한다고 주장하며, 이는 자율적 기계 지능에 대한 Yann LeCun의 비전 [1]과 밀접하게 부합하는 목표이다. 우리는 다음과 같은 근본적 가정에 기반하여 그러한 모델을 개발하고자 한다. 각 질문과 각 모델에 대해, 모델이 정답에 도달하는 데 필요한 최소한의 수로 정의되는 최적의 도구 호출 수가 존재한다는 것이다. 결정적으로, 이 최적의 수는 모델이나 질문에 따라 달라질 수 있다. 같은 질문에 대해서도 추론 능력이나 표현 능력이 다른 모델들은 서로 다른 수준의 도구 도움을 필요로 할 수 있다. 마찬가지로 단일 모델이라 하더라도 필요한 도구 호출의 수는 서로 다른 질문의 복잡도에 따라 크게 달라질 수 있다. 예를 들어, 어떤 질문들은 내부 추론만으로 해결 가능하여 도구 호출이 0번 필요한 반면, 다른 질문들은 정답을 산출하기 위해 한 번 이상의 도구 상호작용을 필요로 할 수 있다. 이러한 가변성은 또한, 천편일률적인 방법(즉, SFT)이 아니라 질문별·모델별로 에이전트의 행동을 조정하는 데 있어 RL이 갖는 고유한 이점을 부각시킨다. 이상적으로는, 같은 질문에 대해 정답으로 이어지는 모든 도구통합추론 궤적들 가운데, RL의 보상 함수는 최적(즉, 최소)의 도구 호출 수를 사용하는 궤적에 가장 높은 보상을 부여해야 한다. 과도한 도구 호출을 수반하는 다른 정답 궤적들은 정의상 차선책이므로 더 낮은 보상을 받아야 한다. 이러한 정식화는 모델이 질문별·모델별로 정답 정확성과 최적의 도구 사용 행동을 함께 최적화하는 정책을 학습할 수 있게 한다.

이어서 우리는 최적 도구 호출 제어 정책 최적화(Optimal Tool Calls controlled Policy Optimization, OTC-PO)를 제안하는데, 이는 모델이 정답을 제공하는 것 외에도 최적의 도구 사용 행동을 학습할 수 있게 하는 단순하지만 효과적인 RL 기반 방법이다. OTC-PO는 정답에 도달하는 데 필요한 도구 호출 수를 최소화하도록 모델을 학습시켜, 정확도를 희생하지 않으면서 학습과 추론 비용을 모두 줄인다. 이를 달성하기 위해 우리는 정확성과 같은 전통적 보상 신호를, 도구 효율성을 반영하는 스케일링 계수로 조절하는 도구통합보상(tool-integrated reward)을 도입한다. 이는 모델이 더 적은 도구 호출을 요하는 정답을 우선시하도록 유도한다. 우리의 정식화는 최적화 목표를 정확성 단독에서 도구 생산성(tool productivity)으로 전환하는데, 이는 과제 이득(예: 정답 정확도)과 도구 사용 비용(예: 도구 호출 수)의 비율로 정의된다. 우리가 아는 한, 이는 도구통합추론의 효율성과 효과성을 모두 명시적으로 최적화하는 선구적인 RL 기반 프레임워크이다. OTC-PO는 경량이며 플러그 앤 플레이 방식으로, 표준 RL 파이프라인에 최소한의 변경만을 요구하여 기존 시스템에 쉽게 도입할 수 있다. 우리는 널리 사용되는 두 가지 도구인 검색과 코드에 대해 여러 LLMs, 즉 Qwen-2.5-3B/7B-Base와 Qwen2.5-Math-1.5B/7B-Base [18]를 기반으로 포괄적인 실험을 수행하여, 강력한 베이스라인과 거의 동등한 정확도를 유지하면서 궤적에서의 도구 호출을 상당히 감소시킨 모델들을 도출한다. 요약하면, 핵심 기여는 다음과 같다.

• 우리는 i) 선행 연구에서 대체로 간과되어 온 새로운 과제 정식화를 제안하여 최종 정답 정확성을 넘어 도구 사용 행동을 고려하고, ii) LLMs의 TIR에서 인지적 부하 이전 현상을 규명하며, iii) TIR의 효과성과 효율성을 측정하기 위한 도구 생산성 개념을 도입하는 선구적인 연구이다.

• 우리는 각 질문-모델 쌍에 대해 필요한 최소한의 도구 호출 수가 존재한다는 근본적 관찰에 기반하여, 모델이 문제를 해결하고 도구 생산성을 극대화하기 위해 최적의 도구 호출을 사용하도록 유도하는, 단순하고 더 빠르며 확장 가능하고 일반화 가능한 OTC-PO 알고리즘을 제안한다. 이는 다양한 RL 알고리즘과 호환되며 쉽게 구현될 수 있음을 밝힌다.

• 우리는 PPO [19]와 GRPO [20] 알고리즘에 기반하여 적응성과 일반화를 잃지 않으면서 OTC-PPO와 OTC-GRPO를 두 가지 대표적 방법으로 구현하며, 여러 벤치마크와 베이스라인에 대한 실험 결과는 도메인 내(in-domain) 및 도메인 외(out-of-domain) 평가 양쪽에서 정확도를 대부분 보존하면서 도구 호출 비용을 유의미하게 감소시킴을 입증한다.
