## 원문 (abstract + intro)

Abstract
Language models (LMs) exhibit remarkable
abilities to solve new tasks from just a few
examples or textual instructions, especially at
scale. They also, paradoxically, struggle with
basic functionality, such as arithmetic or fac-
tual lookup, where much simpler and smaller
models excel.
In this paper, we show that
LMs can teach themselves to use external tools
via simple APIs and achieve the best of both
worlds.
We introduce Toolformer, a model
trained to decide which APIs to call, when to
call them, what arguments to pass, and how to
best incorporate the results into future token
prediction. This is done in a self-supervised
way, requiring nothing more than a handful of
demonstrations for each API. We incorporate
a range of tools, including a calculator, a Q&A
system, a search engine, a translation system,
and a calendar. Toolformer achieves substan-
tially improved zero-shot performance across
a variety of downstream tasks, often competi-
tive with much larger models, without sacriﬁc-
ing its core language modeling abilities.

Our approach for achieving these goals is based
on the recent idea of using large LMs with in-
context learning (Brown et al., 2020) to generate
entire datasets from scratch (Schick and Schütze,
2021b; Honovich et al., 2022; Wang et al., 2022):
Given just a handful of human-written examples
of how an API can be used, we let a LM annotate
a huge language modeling dataset with potential
API calls. We then use a self-supervised loss to
determine which of these API calls actually help
the model in predicting future tokens. Finally, we
ﬁnetune the LM itself on the API calls that it con-
siders useful. As illustrated in Figure 1, through
this simple approach, LMs can learn to control a va-
riety of tools, and to choose for themselves which
tool to use when and how.
As our approach is agnostic of the dataset be-
ing used, we can apply it to the exact same dataset
that was used to pretrain a model in the ﬁrst place.
This ensures that the model does not lose any
of its generality and language modeling abilities.
We conduct experiments on a variety of differ-
ent downstream tasks, demonstrating that after
learning to use tools, Toolformer, which is based
on a pretrained GPT-J model (Wang and Komat-
suzaki, 2021) with 6.7B parameters, achieves much
stronger zero-shot results, clearly outperforming a
much larger GPT-3 model (Brown et al., 2020) and
several other baselines on various tasks.


## 한글 번역

초록

언어모델(language model, LM)은 특히 대규모일수록 단지 몇 개의 예시나 텍스트 지시만으로 새로운 과제를 해결하는 놀라운 능력을 보인다. 그러나 역설적이게도, 훨씬 더 단순하고 작은 모델이 잘 해내는 산술 연산이나 사실 조회와 같은 기본적인 기능에서는 어려움을 겪는다. 본 논문에서 우리는 언어모델이 간단한 API를 통해 외부 도구를 사용하는 법을 스스로 학습하여 두 세계의 장점을 모두 취할 수 있음을 보인다. 우리는 어떤 API를 호출할지, 언제 호출할지, 어떤 인자를 전달할지, 그리고 그 결과를 이후의 토큰 예측에 어떻게 가장 잘 통합할지를 결정하도록 학습된 모델인 Toolformer를 소개한다. 이는 자기지도(self-supervised) 방식으로 이루어지며, 각 API에 대해 소수의 시연 예시만을 필요로 한다. 우리는 계산기, 질의응답(Q&A) 시스템, 검색 엔진, 번역 시스템, 그리고 달력을 포함한 다양한 도구를 통합한다. Toolformer는 다양한 다운스트림 과제 전반에서 제로샷(zero-shot) 성능을 상당히 향상시키며, 핵심적인 언어모델링 능력을 희생하지 않으면서도 종종 훨씬 더 큰 모델과 경쟁할 만한 수준에 도달한다.

1. 서론

대규모 언어모델은 다양한 자연어처리 과제에서 인상적인 제로샷 및 퓨샷(few-shot) 결과를 달성하며(Brown et al., 2020; Chowdhery et al., 2022, i.a.), 여러 창발적 능력을 보인다(Wei et al., 2022). 그러나 이러한 모델들은 모두, 추가적인 규모 확장으로도 기껏해야 부분적으로만 해결될 수 있는 몇 가지 본질적 한계를 지닌다. 이러한 한계에는 최근 사건에 대한 최신 정보에 접근하지 못하는 점(Komeili et al., 2022)과 그와 관련된 사실 환각(hallucinate) 경향(Maynez et al., 2020; Ji et al., 2022), 저자원(low-resource) 언어를 이해하는 데 따르는 어려움(Lin et al., 2021), 정밀한 계산을 수행하는 데 필요한 수학적 능력의 부재(Patel et al., 2021), 그리고 시간의 흐름에 대한 인식 부족(Dhingra et al., 2022) 등이 포함된다.

오늘날의 언어모델이 지닌 이러한 한계를 극복하는 간단한 방법은, 모델에게 검색 엔진, 계산기, 달력과 같은 외부 도구를 사용할 수 있는 능력을 부여하는 것이다. 그러나 기존 접근법들은 대량의 인간 주석에 의존하거나(Komeili et al., 2022; Thoppilan et al., 2022), 도구 사용을 과제별(task-specific) 설정으로만 제한하여(예: Gao et al., 2022; Parisi et al., 2022) 언어모델에서 도구 사용이 보다 폭넓게 채택되는 것을 가로막는다. 따라서 우리는 다음의 요건을 충족하는 새로운 방식으로 도구 사용을 학습하는 모델인 Toolformer를 제안한다.

- 도구 사용은 대량의 인간 주석을 필요로 하지 않는 자기지도 방식으로 학습되어야 한다. 이는 그러한 주석에 수반되는 비용 때문일 뿐만 아니라, 인간이 유용하다고 여기는 것이 모델이 유용하다고 여기는 것과 다를 수 있기 때문에 중요하다.

- 언어모델은 자신의 범용성을 전혀 잃지 않아야 하며, 언제 어떻게 어떤 도구를 사용할지를 스스로 결정할 수 있어야 한다. 기존 접근법과 달리, 이는 특정 과제에 얽매이지 않는 훨씬 더 포괄적인 도구 사용을 가능하게 한다.

이러한 목표를 달성하기 위한 우리의 접근법은, 대규모 언어모델을 문맥 내 학습(in-context learning)(Brown et al., 2020)과 함께 사용하여 전체 데이터셋을 처음부터 생성한다는 최근의 아이디어(Schick and Schütze, 2021b; Honovich et al., 2022; Wang et al., 2022)에 기반한다. 즉, API가 어떻게 사용될 수 있는지에 대한 소수의 인간이 작성한 예시만 주어지면, 우리는 언어모델로 하여금 거대한 언어모델링 데이터셋에 잠재적인 API 호출을 주석으로 달게 한다. 그런 다음 자기지도 손실을 사용하여 이러한 API 호출 중 실제로 모델이 이후 토큰을 예측하는 데 도움이 되는 것이 무엇인지 판별한다. 마지막으로, 모델 스스로가 유용하다고 판단한 API 호출에 대해 언어모델 자체를 미세조정(finetune)한다. Figure 1에 나타난 것처럼, 이러한 간단한 접근법을 통해 언어모델은 다양한 도구를 제어하고, 언제 어떻게 어떤 도구를 사용할지를 스스로 선택하는 법을 학습할 수 있다.

우리의 접근법은 사용되는 데이터셋에 구애받지 않으므로, 애초에 모델을 사전학습하는 데 사용된 바로 그 데이터셋에 동일하게 적용할 수 있다. 이는 모델이 자신의 범용성과 언어모델링 능력을 전혀 잃지 않도록 보장한다. 우리는 다양한 다운스트림 과제에 대해 실험을 수행하여, 도구 사용을 학습한 이후의 Toolformer가, 67억 개의 파라미터를 가진 사전학습된 GPT-J 모델(Wang and Komatsuzaki, 2021)에 기반함에도 불구하고, 훨씬 더 강력한 제로샷 결과를 달성하여 여러 과제에서 훨씬 더 큰 GPT-3 모델(Brown et al., 2020)과 그 외 여러 베이스라인을 분명하게 능가함을 입증한다.
