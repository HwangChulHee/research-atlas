## 원문 (abstract + intro)

Abstract
Standard Retrieval-Augmented Generation (RAG) relies on chunk-based retrieval,
whereas GraphRAG advances this approach by graph-based knowledge representa-
tion. However, existing graph-based RAG approaches are constrained by binary
relations, as each edge in an ordinary graph connects only two entities, limiting
their ability to represent the n-ary relations (n ≥2) in real-world knowledge. In
this work, we propose HyperGraphRAG, the first hypergraph-based RAG method
that represents n-ary relational facts via hyperedges. HyperGraphRAG consists of
a comprehensive pipeline, including knowledge hypergraph construction, retrieval,
and generation. Experiments across medicine, agriculture, computer science, and
law demonstrate that HyperGraphRAG outperforms both standard RAG and pre-
vious graph-based RAG methods in answer accuracy, retrieval efficiency, and
generation quality. Our data and code are publicly available1.

the fact that “Male hypertensive patients with serum creatinine levels between 115–133 µmol/L are
diagnosed with mild serum creatinine elevation” requires decomposing it into several binary relational
triples, such as Gender:(Hypertensive patient, Male) and Diagnosed_with:(Hypertensive patient,

To address these limitations, we propose HyperGraphRAG, as illustrated in Figure 1, a novel graph-
based RAG method built upon hypergraph-structured knowledge representation. In contrast
to prior graph-based RAG methods constrained to binary relations, HyperGraphRAG leverages
hyperedges to represent n-ary relational facts, where each hyperedge connects n entities (n ≥2),
e.g. Hyperedge:(Hypertensive patient, Male, Serum creatinine levels between 115–133 µmol/L, Mild
serum creatinine elevation), and each hyperedge is expressed through natural language descriptions.
This design ensures knowledge completeness, structural expressiveness, and inferential capability,
thereby providing more comprehensive support for knowledge-intensive applications.
Our proposed HyperGraphRAG is built upon three key steps. First, we propose a knowledge
hypergraph construction method, leveraging LLM-based n-ary relation extraction to extract and
structure multi-entity relationships. The resulting hypergraph is stored in a bipartite graph database,
with separate vector databases for entities and hyperedges to facilitate efficient retrieval. Second, we
develop a hypergraph retrieval strategy that employs vector similarity search to retrieve relevant
entities and hyperedges, ensuring that the knowledge retrieved is both precise and contextually
relevant. Lastly, we introduce a hypergraph-guided generation mechanism, which combines
retrieved n-ary facts with traditional chunk-based RAG passages, thereby improving response quality.
To validate the effectiveness, we conduct experiments in multiple knowledge-intensive domains [7], in-
cluding medicine, agriculture, computer science, and law. Results demonstrate that HyperGraphRAG
outperforms standard RAG and previous graph-based RAG methods in answer accuracy, retrieval
efficiency, and generation quality, showcasing its strong potential for real-world applications.


## 한글 번역

초록

표준 검색증강생성(RAG)은 청크 기반(chunk-based) 검색에 의존하는 반면, GraphRAG는 그래프 기반 지식 표현으로 이 접근법을 발전시킨다. 그러나 기존의 그래프 기반 RAG 접근법들은 이진 관계(binary relations)에 제약되는데, 일반적인 그래프에서 각 엣지(edge)는 두 개의 개체만 연결하기 때문에 실세계 지식에 존재하는 n항 관계(n-ary relations, n ≥ 2)를 표현하는 능력이 제한된다. 본 연구에서 우리는 하이퍼엣지(hyperedge)를 통해 n항 관계 사실을 표현하는 최초의 하이퍼그래프 기반(hypergraph-based) RAG 방법인 HyperGraphRAG를 제안한다. HyperGraphRAG는 지식 하이퍼그래프 구성, 검색, 생성을 포함하는 종합적 파이프라인으로 구성된다. 의학, 농업, 컴퓨터 과학, 법학에 걸친 실험은 HyperGraphRAG가 답변 정확도, 검색 효율성, 생성 품질에서 표준 RAG와 이전의 그래프 기반 RAG 방법들을 모두 능가함을 입증한다. 우리의 데이터와 코드는 공개적으로 이용 가능하다1.

그림 1: HyperGraphRAG의 예시.

검색증강생성(RAG) [10, 6]은 지식 검색을 대규모 언어 모델(LLM) [17, 28]과 통합함으로써 지식 집약적 작업을 발전시켜, 사실 인식과 생성 정확도를 강화해 왔다. 표준 RAG는 일반적으로 청크 기반 검색에 의존하여, 문서를 고정 길이의 텍스트 청크로 분할하고 밀집 벡터 유사도(dense vector similarity)를 통해 검색하는데, 이는 개체들 사이의 관계를 간과한다. 최근, GraphRAG [2]는 개체 간 관계를 포착하기 위해 지식을 그래프로 구조화하는 유망한 방향으로 부상했으며, 검색 효율성과 지식 주도 생성을 개선할 잠재력을 지닌다 [18].

그러나 일반적인 그래프에서 각 엣지는 두 개의 개체만 연결하기 때문에, 기존의 그래프 기반 RAG 접근법들 [2, 7, 1, 8]은 모두 이진 관계에 제약되어, 실세계 도메인 지식에 널리 퍼져 있는 둘 이상의 개체 간 n항 관계를 모델링하기에 불충분하다 [25]. 예를 들어, 의료 도메인에서 그림 2에 예시된 바와 같이,

"혈청 크레아티닌 수치가 115~133 μmol/L 사이인 남성 고혈압 환자는 경미한 혈청 크레아티닌 상승으로 진단된다"는 사실을 표현하려면, 이를 성별:(고혈압 환자, 남성)과 진단됨:(고혈압 환자, 경미한 혈청 크레아티닌 상승)과 같은 여러 이진 관계 삼중쌍(triple)으로 분해해야 하며, 이는 변환 과정에서 표현 희소성(representation sparsity)을 초래한다.

이러한 한계를 해결하기 위해, 우리는 그림 1에 예시된 바와 같이 하이퍼그래프 구조 지식 표현 위에 구축된 새로운 그래프 기반 RAG 방법인 HyperGraphRAG를 제안한다. 이진 관계에 제약된 이전의 그래프 기반 RAG 방법들과 대조적으로, HyperGraphRAG는 하이퍼엣지를 활용하여 n항 관계 사실을 표현하는데, 여기서 각 하이퍼엣지는 n개의 개체를 연결하며(n ≥ 2), 예를 들어 하이퍼엣지:(고혈압 환자, 남성, 혈청 크레아티닌 수치 115~133 μmol/L, 경미한 혈청 크레아티닌 상승)와 같고, 각 하이퍼엣지는 자연어 기술을 통해 표현된다. 이 설계는 지식 완전성, 구조적 표현력, 추론 능력을 보장하여, 지식 집약적 응용에 더 종합적인 지원을 제공한다.

우리가 제안하는 HyperGraphRAG는 세 가지 핵심 단계 위에 구축된다. 첫째, 우리는 LLM 기반 n항 관계 추출을 활용하여 다중 개체 관계를 추출하고 구조화하는 지식 하이퍼그래프 구성 방법을 제안한다. 결과로 생성된 하이퍼그래프는 이분 그래프 데이터베이스(bipartite graph database)에 저장되며, 효율적 검색을 촉진하기 위해 개체와 하이퍼엣지를 위한 별도의 벡터 데이터베이스를 둔다. 둘째, 우리는 벡터 유사도 검색을 사용하여 관련 개체와 하이퍼엣지를 검색하는 하이퍼그래프 검색 전략을 개발하여, 검색된 지식이 정밀하고 맥락적으로 관련성이 있도록 보장한다. 마지막으로, 우리는 검색된 n항 사실을 전통적인 청크 기반 RAG 구절과 결합하는 하이퍼그래프 유도 생성(hypergraph-guided generation) 메커니즘을 도입하여, 응답 품질을 개선한다.

효과성을 검증하기 위해, 우리는 의학, 농업, 컴퓨터 과학, 법학을 포함한 다수의 지식 집약적 도메인 [7]에서 실험을 수행한다. 결과는 HyperGraphRAG가 답변 정확도, 검색 효율성, 생성 품질에서 표준 RAG와 이전의 그래프 기반 RAG 방법들을 능가함을 입증하여, 실세계 응용을 위한 그 강력한 잠재력을 보여준다.
